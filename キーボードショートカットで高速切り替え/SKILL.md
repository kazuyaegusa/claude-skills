# キーボードショートカットで高速切り替え

> Superset UIに組み込まれたカスタマイズ可能なショートカット（⌘1-9でワークスペース切替、⌘Tで新規タブ、⌘Dで画面分割等）を活用し、マウスなしでエージェント間を瞬時に移動する

- 出典: https://github.com/superset-sh/superset
- 投稿者: superset-sh
- カテゴリ: agent-orchestration

## なぜ使うのか

10個のエージェントを監視する際、マウスでクリックしていると時間がかかり集中が途切れる。ショートカットならキーボードだけで視点を移動でき、レビュー→編集→次のエージェントへの流れが高速化する。

## いつ使うのか

複数ワークスペースを頻繁に行き来する時、エージェント完了通知が連続して届いた時

## やり方

1. Supersetで `⌘/` を押して設定画面を開く
2. Keyboard Shortcuts セクションで各操作のキーを確認・変更
3. ワークスペース切り替えは `⌘1-9` または `⌘⌥↑/↓`
4. 変更パネルの表示切替は `⌘L`
5. 外部エディタで開くは `⌘O`
6. ターミナル操作は `⌘T`（新規タブ）、`⌘D`（右分割）、`⌘⇧D`（下分割）
7. プリセット起動は `Ctrl+1-9`
8. すべてドキュメント（https://docs.superset.sh/keyboard-shortcuts）に記載

### 入力

- Supersetのデフォルトまたはカスタムショートカット設定

### 出力

- キーボードのみでの高速ナビゲーション
- コンテキストスイッチ時間の最小化

## 使うツール・ライブラリ

- Superset内蔵ショートカットエンジン

## 前提知識

- git 2.20以上がインストールされていること（worktree機能）
- macOS環境（Windows/Linuxは未検証）
- Bun v1.0+ランタイム（ビルド・実行用）
- GitHub CLI（gh）とCaddy（開発サーバー用）のインストール
- CLIベースのAIエージェント（Claude Code, Codex, Gemini CLI等）が動作する環境
- git worktreeの基本概念（複数の作業ディレクトリを同一リポジトリで管理）の理解

## 根拠

> ".superset/config.json" で "setup": ["./.superset/setup.sh"] を定義することでワークスペース作成時に自動実行
