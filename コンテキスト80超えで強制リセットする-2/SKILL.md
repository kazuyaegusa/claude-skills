# コンテキスト80%超えで強制リセットする

> ccslのステータスラインでコンテキスト使用率が80%を超えたら、その時点でセッションをリセット（圧縮または新規開始）するルールを運用として設ける。

- 出典: https://x.com/masahirochaen/status/2038735267117346923
- 投稿者: チャエン | デジライズ CEO《重要AIニュースを毎日最速で発信》
- カテゴリ: context-management

## なぜ使うのか

コンテキストが肥大化するほど古い情報が参照されにくくなり、回答精度が落ちる。80%を閾値に機械的にリセットすることで、感覚に頼らず一貫した品質を維持できる。

## いつ使うのか

ccslのコンテキストバーが黄色（80%以上）に変わったとき。長時間セッション中にClaude の回答がずれてきたと感じたとき。

### 具体的な適用場面

- 1時間以上のデバッグや機能実装セッションで、途中からClaude の回答精度が落ちてきた気がするとき
- コード量の多いリポジトリでClaude Codeを使っており、コンテキスト超過エラーや品質ムラが頻発しているとき

## やり方

1. ccslのステータスラインを常時確認し、2行目のプログレスバーが黄色（80%）になったらリセットのタイミングと判断
2. リセット方法はどちらか選択：
   - コンテキスト圧縮: Claude Code内で `/compact` を実行（重要な文脈を保持しつつ圧縮）
   - 新規セッション開始: Claude Codeを再起動して完全にクリアにする
3. 赤色警告（90%）まで放置しない

### 入力

- ccslインストール済みのClaude Code環境
- コンテキスト使用率80%以上の状態

### 出力

- コンテキストがリセットされた新鮮な状態のセッション
- 精度劣化のない回答品質の維持

## 使うツール・ライブラリ

- ccsl
- Claude Code /compact コマンド

## コード例

```
# Claude Code内で実行
/compact
# または新規セッション起動でコンテキストをクリア
```

## 前提知識

- Claude Codeがインストール済みであること
- Python 3.9以上（pip経由でインストールする場合）
- macOS（brew経由の場合）
