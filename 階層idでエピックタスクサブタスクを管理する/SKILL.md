# 階層IDでエピック・タスク・サブタスクを管理する

> Beadsはハッシュベースの階層ID（`bd-a3f8`, `bd-a3f8.1`, `bd-a3f8.1.1`）を使い、エピック配下のタスク・サブタスクを構造化して管理する

- 出典: https://github.com/steveyegge/beads
- 投稿者: steveyegge
- カテゴリ: agent-orchestration

## なぜ使うのか

大きな機能追加（エピック）を細分化し、依存関係を明示することで、エージェントがどのタスクから着手すべきか自動判定できる

## いつ使うのか

大規模な機能開発やリファクタリングをエージェントに段階的に実行させたい場合

## やり方

1. エピック作成: `bd create "新機能開発" -p 0`（例: bd-a3f8が発番）
2. 子タスク作成時に親IDを指定（内部的に階層IDが付与される）
3. `bd ready` で実行可能タスク（ブロッカーなし）を取得
4. `bd show <id>` で依存関係と監査ログを確認

### 入力

- エピックタイトル
- 子タスク・サブタスクの依存関係

### 出力

- 階層構造を持つタスクグラフ
- 実行可能タスクリスト（bd ready）

## 使うツール・ライブラリ

- Beads CLI

## コード例

```
bd create "新認証システム" -p 0  # bd-a3f8
bd create "OAuth実装" -p 1 --parent bd-a3f8  # bd-a3f8.1
bd create "トークン検証" -p 1 --parent bd-a3f8.1  # bd-a3f8.1.1
bd ready  # ブロッカーのないタスクを表示
```

## 前提知識

- AIコーディングエージェント（Claude Code, Cursor等）の基本的な使い方
- コマンドラインツールのインストール方法（curl, npm, Homebrew等）
- Gitの基本操作（任意だがあると便利）
- SQLデータベースの概念（Doltが内部で使われる）
- 依存グラフ・DAGの概念（タスク管理に必要）
