# コンテキスト80%超えで強制リセットする

> ccslのコンテキスト使用率表示を監視し、80%を超えた時点でClaude Codeセッションを手動でリセットする運用ルールを設ける

- 出典: https://x.com/masahirochaen/status/2038735267117346923
- 投稿者: チャエン | デジライズ CEO《重要AIニュースを毎日最速で発信》
- カテゴリ: claude-code-workflow

## なぜ使うのか

コンテキストが肥大すると過去の会話履歴のノイズが増え、LLMの出力精度が低下する。しきい値を80%に設定することで精度劣化が顕在化する前に予防的にリセットできる

## いつ使うのか

ステータスラインのコンテキスト使用率が80%を超えたとき、または出力品質の低下を感じ始めたとき

### 具体的な適用場面

- 1時間以上続く大規模コーディングセッションで精度劣化を感じ始めたとき
- 複数ファイルにまたがるリファクタリング・機能追加で多数のツール呼び出しが発生するとき
- 週次コスト・トークン消費を管理しながらClaude Codeを運用するとき

## やり方

1. ccslのステータスラインでコンテキスト使用率を確認する（`Context: [XX%]` の表示）
2. XX%が80を超えたら `/clear` コマンドまたはセッション再起動でコンテキストをリセットする
3. ccslは80%で黄色、90%で赤色の警告色変化を自動表示するため、視覚的なアラートとして機能する
4. リセット前に必要なコンテキスト（タスク定義・現在の状態）を別ファイルに記録しておき、リセット後に `@ファイル名` で再注入する

### 入力

- ccslがインストールされコンテキスト使用率が表示されている状態

### 出力

- コンテキストがリセットされた新鮮なセッション。以降の応答精度が回復する

## 使うツール・ライブラリ

- ccsl (コンテキスト使用率の視覚的モニタリング)

## 前提知識

- Claude Codeがインストール・起動できる環境
- Python 3.9以上（pip経由でインストールする場合）
- macOS（Homebrew経由の場合）またはLinux/Windowsでpipが利用可能な環境
