# KEWORK/AIXEED 業務メッセージ下書きスキル

## 概要

KEWORK/AIXEEDの業務コミュニケーションで必要なメッセージを下書きする。
返信メール、LINEメッセージ、フォローアップ連絡など、実業務で発生する文面を即座に生成する。

## 実行手順

### 1. 入力情報の確認

タスク文面から以下を特定する:

- **宛先**: 誰に送るか（個人名、参加者グループなど）
- **チャネル**: メール / LINE / 公式LINE / Slack / その他
- **目的**: お礼 / フォローアップ / 問い合わせ回答 / 連絡・報告 / 依頼
- **コンテキスト**: 何についてのメッセージか（説明会、コース申込、決済完了など）

情報が不足している場合は、推測で埋めず「不明」と明記して下書きを出す。

### 2. トーンガイドライン（KEWORK/AIXEEDスタイル）

谷内さんのコミュニケーションスタイルに準拠する:

- **丁寧だが堅すぎない**: 「です・ます」調だが、ビジネス定型文の羅列にしない
- **温かみがある**: 相手への感謝や気遣いを自然に入れる
- **簡潔で要件が明確**: 長文にせず、何をしてほしいか・何を伝えたいかが一目でわかる
- **人間味がある**: 機械的にならない。相手の状況に触れる一文を入れる
- **敬語レベル**: 初対面・外部 → 丁寧語、既存メンバー → ややカジュアルな丁寧語
- **絵文字**: LINE → 適度に使用可、メール → 使用しない

### 3. メッセージテンプレート

#### A. お礼メッセージ（説明会参加、面談後など）

```
件名: 【KEWORK】○○へのご参加ありがとうございました

{相手の名前}様

先日は{イベント名}にご参加いただき、ありがとうございました。
{イベントでの相手の反応や印象に触れる一文}

{次のアクションや共有情報}

ご不明な点がございましたら、お気軽にご連絡ください。
今後ともよろしくお願いいたします。

KEWORK / AIXEED
{送信者名}
```

#### B. フォローアップメッセージ（申込後、説明会後のアクション促進）

```
件名: 【KEWORK】{コース名/イベント名}について

{相手の名前}様

{イベント名}へのご参加（ご関心）ありがとうございます。

{フォローアップの本題: 次のステップ、必要な手続き、期限など}

▼ 手順
1. {ステップ1}
2. {ステップ2}
3. {ステップ3}

ご質問があれば、このままご返信（LINE返信）いただければ対応いたします。

KEWORK / AIXEED
{送信者名}
```

#### C. 問い合わせ回答（メールアドレス提出方法、コース詳細など）

```
件名: Re: {元の件名}

{相手の名前}様

お問い合わせいただきありがとうございます。
{質問内容}について、ご案内いたします。

{回答内容を簡潔に}

{補足情報があれば}

他にもご不明な点がございましたら、お気軽にお問い合わせください。

KEWORK / AIXEED
{送信者名}
```

#### D. 連絡・報告（決済完了、コース決定、進捗共有など）

```
件名: 【KEWORK】{件名: 決済完了のご連絡 / コース選択について}

{相手の名前}様

{報告の本題を1文で}

{詳細・補足}

{次のステップがあれば記載}

引き続きよろしくお願いいたします。

KEWORK / AIXEED
{送信者名}
```

#### E. LINEメッセージ（公式LINE送信用、短縮版）

```
{相手の名前}さん、{イベント名}へのご参加ありがとうございました！

{要件を2-3行で}

{アクションがあれば}

ご質問はこのLINEにそのままお送りください。
```

### 4. 出力フォーマット

必ず以下の形式で出力する:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
送信先: {宛先の名前・グループ}
チャネル: {メール / LINE / 公式LINE / Slack}
目的: {お礼 / フォローアップ / 問い合わせ回答 / 連絡・報告}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

件名: {件名（メールの場合）}

{本文}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
確認事項:
- [ ] 宛先は正しいか
- [ ] チャネル（メール/LINE）は正しいか
- [ ] 送信タイミングは適切か
- [ ] 添付ファイルは必要か
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 5. 注意事項

- 相手の名前が不明な場合は `{相手の名前}` のままプレースホルダーとして残す
- 金額・日時・場所など事実情報は、タスク文面に含まれていない限り推測で埋めない。`{要確認: 金額}` のように明示する
- 複数人への一斉送信の場合は、個別カスタマイズが必要な箇所を明記する
- FUSION AIなど外部パートナーへの連絡は、よりフォーマルなトーンにする
- 九州オフ会など社内イベントの情報共有は、ややカジュアルでOK
