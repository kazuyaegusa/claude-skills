# サブエージェント駆動開発でタスクを並列実行する

> 実装計画を最小サブタスク単位に分割し、各タスクをサブエージェントに委任して並列実行する。各エージェントの作業を監督・検査しながら進める

- 出典: https://x.com/jiroucaigou/status/2035964407201849564
- 投稿者: 努力赚钱的菜狗
- カテゴリ: agent-orchestration

## なぜ使うのか

単一の長いコンテキストでAIに複雑なタスクを処理させると誤りが蓄積する。最小単位のタスクに分割することでコンテキスト汚染を防ぎ、並列実行で全体の処理時間を短縮できる

## いつ使うのか

複数の独立したサブタスクに分解できるコーディングタスクがあるとき。特にエンジニアリングタスクが3つ以上あるプロジェクト

### 具体的な適用場面

- Claude Code / Cursorで新機能を実装しようとするたびにAIが設計なしに書き始めて後悔する状況
- チームでAIコーディングを使っているが、各人がバラバラなプロンプト戦略を取っていて品質が安定しない状況
- AIが生成したコードが動くが後のリファクタ・バグ修正が困難になっているプロジェクト

## やり方

1. SuperPowersのsubagent-driven-developmentスキルが承認済み設計文書から実装計画を生成
2. 計画を最小サブタスクに分解（TDD準拠: テスト先行）
3. `claude -p` またはAgent toolで各タスクをサブエージェントに委任
4. 各サブエージェントが完了したらsuper agentが作業内容を検査・レビューし次タスクへ
5. ユーザーは全体を監督するだけでよい（「コーヒーを飲みながら監督」）

### 入力

- 承認済み設計文書
- 実装計画（SubTaskリスト）

### 出力

- 各サブタスクの実装コード
- テスト通過済みの成果物

## 使うツール・ライブラリ

- obra/superpowers (subagent-driven-developmentスキル)
- claude -p

## 前提知識

- Claude Code または Cursor が動作する環境
- Node.js / npx が使える環境（CLIインストールの場合）
- git が使える環境（worktree機能のため）
- AIコーディングエージェントの基本的な使い方を知っていること

## 根拠

> 「SuperPowers制定了一套规则，让AI可以像资深团队一样有一套标准的开发流程。目前已在Github斩获106kstars」

> 「1.先疯狂提问，把你的需求完全理解，你同意后并生成标准设计文档」

> 「2.开新分支+隔离工作区，确保不会搞乱主代码」

> 「3.把任务拆分成最小子任务」

> 「4.交给子Agent执行并强制测试，最小代码也要通过测试」
