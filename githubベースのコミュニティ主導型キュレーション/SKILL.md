# GitHubベースのコミュニティ主導型キュレーション

> awesome-*形式のGitHubリポジトリを作成し、Pull Request経由で誰でもスキルを追加できる仕組みを構築

- 出典: https://github.com/BehiSecc/awesome-claude-skills
- 投稿者: BehiSecc
- カテゴリ: other

## なぜ使うのか

中央集権的な管理ではなく、コミュニティ全体で知見を蓄積することで、カタログの鮮度・網羅性・品質が継続的に向上する

## いつ使うのか

専門知識が分散しており、1人では全領域をカバーできない時

## やり方

1. GitHubでawesome-claude-skillsリポジトリを作成
2. CONTRIBUTION.mdで追加手順を明記（Fork → 編集 → Pull Request）
3. Issueテンプレートで不具合報告・改善提案を受付
4. 定期的にPRをレビュー・マージしてカタログを更新

### 入力

- コミュニティからのPull Request
- 新規スキルのGitHubリポジトリ・説明

### 出力

- 常に最新のスキルカタログ
- コミュニティ貢献者のリスト

## 使うツール・ライブラリ

- GitHub
- Pull Request workflow

## 前提知識

- Claude Codeの基本的な使い方
- GitHubの基本操作（clone, fork, pull request）
- Markdownの記法
- 各スキルが対象とする技術領域の基礎知識（例：PDF処理ならPyMuPDF, セキュリティならOWASP Top 10等）

## 根拠

> 1. Fork this repo 2. Make your changes 3. Submit a Pull Request - コミュニティ貢献手順
