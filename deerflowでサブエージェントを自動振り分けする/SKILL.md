# DeerFlowでサブエージェントを自動振り分けする

> 入力タスクの内容に応じて、適切なサブエージェント（検索・コード実行・分析など）を自動選択してルーティングする

- 出典: https://x.com/dify_base/status/2031190239885353023
- 投稿者: AX Base| AX情報発信メディア
- カテゴリ: agent-orchestration

## なぜ使うのか

人間が毎回「どのエージェントに頼むか」を判断するコストを排除し、複合タスクを1つの指示で完結させるため

## いつ使うのか

「検索してコードを書いてレポートを作る」のような複合ステップのタスクを一括で自律処理させたいとき

### 具体的な適用場面

- 複数の専門エージェント（コード生成・検索・分析）を自動選択して一連のリサーチタスクを処理したいとき
- AIエージェントにコードを実行させる必要があるが、ホスト環境を汚染したくないとき
- 複数セッションにまたがる長期プロジェクトで文脈を保持しながらエージェントを動かしたいとき

## やり方

1. `git clone https://github.com/bytedance/DeerFlow` でリポ取得
2. 設定ファイルでサブエージェントの種類と対応タスクパターンを定義
3. メインエージェントにタスクを投入すると、内部でルーターがサブエージェントを選択して実行
※具体的な設定スキーマはリポのREADMEまたはdocs/を参照（投稿に記載なし）

### 入力

- タスク指示文
- サブエージェント定義設定

### 出力

- 各サブエージェントが連携して生成した最終成果物

## 使うツール・ライブラリ

- DeerFlow (ByteDance OSS)

## 前提知識

- AIエージェントの基本概念（LLM・ツール呼び出し・エージェントループ）
- Pythonまたはエージェントフレームワークの基礎
- Dockerまたはコンテナの基礎（サンドボックス利用時）
- Claude Code または Anthropic API の利用環境（Claude Code連携時）
