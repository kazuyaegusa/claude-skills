# エージェント状態一元モニタリング

> 複数エージェントの実行状態（稼働中/待機中/レビュー待ち）を1画面で監視し、通知でレビュータイミングを把握する

- 出典: https://github.com/superset-sh/superset
- 投稿者: superset-sh
- カテゴリ: agent-orchestration

## なぜ使うのか

並列実行中のエージェントを個別監視すると見逃しが発生し、完了したタスクが放置される

## いつ使うのか

3つ以上のエージェントを同時稼働させる場合、複数プロジェクトを横断管理する場合

## やり方

1. SupersetのUIで全ワークスペースを一覧表示
2. 各エージェントのステータスをリアルタイム監視
3. 変更完了時に通知を受け取る
4. built-in diff viewerで差分を即座に確認
5. 承認またはフィードバックをエージェントに返す

### 入力

- 実行中のエージェントインスタンス
- 各エージェントの出力ログ・状態情報

### 出力

- エージェント状態の可視化ダッシュボード
- レビュー待ちタスクの通知
- 統合差分ビュー

## 使うツール・ライブラリ

- Superset UI
- Electron (デスクトップアプリ基盤)
- tRPC (フロントエンド・バックエンド通信)

## 前提知識

- git worktreeの基本概念（メインリポジトリと独立した作業ツリー）
- CLIベースのAIコーディングエージェントの使用経験
- macOS環境（現時点でWindows/Linuxは未検証）
- Bun v1.0+、git 2.20+、gh CLI、Caddyのインストール

## 根拠

> 「Built-in Diff Viewer: Inspect and edit agent changes without leaving the app」
