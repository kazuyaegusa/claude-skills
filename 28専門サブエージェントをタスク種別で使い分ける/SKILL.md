# 28専門サブエージェントをタスク種別で使い分ける

> フロントエンド・バックエンド・テスト・DB・セキュリティ等の専門サブエージェントを、タスク内容に応じてClaude Codeが自動的に選択・連携させる

- 出典: https://x.com/l_go_mrk/status/2037489559186145787
- 投稿者: AI駆動塾
- カテゴリ: claude-code-workflow

## なぜ使うのか

汎用エージェント1つに全タスクを投げると、知識の深さが平均化される。専門化された28エージェントを分業させることで、各ドメインの深い知識を持つエージェントが担当タスクを処理し、全体品質が向上する

## いつ使うのか

フロントエンド実装・バックエンドAPI・テスト作成・セキュリティ検証を同時に進めたい複合タスクのとき

### 具体的な適用場面

- 新規プロジェクト立ち上げ時に、Claude Codeの能力を最大化した状態でスタートしたいとき
- 既存プロジェクトに対して脆弱性チェックを自動化したいとき（OWASP等102ルール）
- フロントエンド・バックエンド・テスト・セキュリティ等を同時進行で進めたいマルチドメインタスクのとき

## やり方

1. ECCをnpx ecc-universalで導入（28エージェントのYAML定義が~/.claude/agents/以下に配置される）
2. Claude Codeに複合タスクを投げる（例：「認証機能を追加してテストも書いて」）
3. オーケストレーターが内容を解析し、auth-agent・test-agent等を自動選択してサブタスクを割り当てる
4. 各エージェントが独立したコンテキストで並列実行し、結果をマージする

### 入力

- ECCインストール済みのClaude Code環境
- 複数ドメインにまたがる開発タスクの要件

### 出力

- 各専門エージェントが担当領域を処理した結果（コード・テスト・レポート）
- オーケストレーターによる統合サマリー

## 使うツール・ライブラリ

- ecc-universal（npm package）
- Claude Code CLI
- Agent SDK

## コード例

```
# ~/.claude/agents/ 以下のエージェント定義例（YAMLフロントマター）:
# ---
# name: security-agent
# description: セキュリティレビュー・脆弱性検出専門
# tools: [Read, Grep, Bash]
# ---
```

## 前提知識

- Claude Code CLIのインストール（npm install -g @anthropic-ai/claude-code または brew install claude）
- Node.js v18以上（npx実行のため）
- Claude ProまたはMax サブスクリプション（サブエージェント並列実行にはトークン消費が多い）
