# エージェント監視と通知による待機時間削減

> 複数エージェントの実行状態を統合ダッシュボードで監視し、レビュー準備完了時に通知を受け取ることで、人間の待機時間をゼロにする

- 出典: https://github.com/superset-sh/superset
- 投稿者: superset-sh
- カテゴリ: agent-orchestration

## なぜ使うのか

エージェントが10分かかるタスクを実行中、別のエージェントも並列実行していると、どちらが完了したか監視する必要がある。通知がなければポーリングで時間を浪費するか、完了を見逃して遅延が発生する

## いつ使うのか

3つ以上のエージェントを並列実行している時、またはエージェントの実行時間が5分を超える時

## やり方

1. Supersetのワークスペースサイドバー（⌘B）で全エージェントの状態を一覧表示
2. エージェントが変更を生成すると自動的にステータスが更新される
3. ⌘Lで変更パネルを開き、built-in diff viewerで差分を確認
4. ⌘1-9で各ワークスペースに即座に切り替え、コンテキストスイッチングのオーバーヘッドなしでレビュー
5. 承認後は⌘Oで外部IDE（VSCode等）を開いて最終調整

### 入力

- 並列実行中の複数エージェントプロセス
- 各エージェントが生成したgit diff

### 出力

- リアルタイムステータス更新
- レビュー準備完了通知
- unified diff表示

## 使うツール・ライブラリ

- Superset monitoring dashboard
- Supersetのbuilt-in diff viewer

## コード例

```
# キーボードショートカット（カスタマイズ可能）
⌘1-9: ワークスペース切り替え
⌘⌥↑/↓: 前後のワークスペースへ移動
⌘L: 変更パネルトグル
⌘B: ワークスペースサイドバートグル
⌘O: 外部アプリで開く
```

## 前提知識

- Git 2.20以上（worktree機能）
- macOS環境（Windows/Linux未検証）
- Bun v1.0以上（ランタイム）
- GitHub CLI（gh）
- 使用したいCLIエージェント（Claude Code、Codex等）のインストールと認証設定
- git worktreeの基本概念（ブランチとディレクトリの分離）
