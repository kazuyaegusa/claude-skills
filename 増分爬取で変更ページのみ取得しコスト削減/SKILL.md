# 増分爬取で変更ページのみ取得しコスト削減

> 前回爬取時と比較し、更新されたページのみ再取得。未変更ページはスキップ

- 出典: https://x.com/xiaohu/status/2031921663751946646
- 投稿者: 小互
- カテゴリ: other

## なぜ使うのか

全ページを毎回爬取するとAPI呼び出し回数・帯域・時間が膨大。増分爬取により必要な部分のみ取得し、コストと時間を大幅削減

## いつ使うのか

定期的にサイト全体を監視したいが、毎回全ページ爬取するとコストが高い時

### 具体的な適用場面

- 競合サイトの価格・在庫情報を定期的に収集したい時
- ニュースサイト・ブログ全体をアーカイブしたい時
- React/Vue等のSPAサイトから構造化データを抽出したい時
- 変更検知機能を実装したいが、毎回全ページ爬取するとコストが高い時
- HTML解析を避け、直接JSON形式でデータ取得したい時

## やり方

1. 初回爬取時、各ページのハッシュ値またはタイムスタンプをCloudflare側が記録
2. 2回目以降の爬取時、増分モードを指定してAPIコール
3. Cloudflareは各ページをチェックし、変更検知されたページのみ返す
4. 変更なしページは結果に含まれず、課金対象外

### 入力

- 前回爬取時の状態（Cloudflare側で管理）
- 増分モード有効化オプション

### 出力

- 変更されたページのみのHTML/JSON
- 変更検知レポート

## 使うツール・ライブラリ

- Cloudflare Scraping API（増分爬取機能）

## 前提知識

- Cloudflareアカウントとアクティブなサブスクリプション（Scraping API利用可能プラン）
- HTTP APIの基本的な使い方（POST/GET、認証ヘッダー設定）
- JSONスキーマの基本理解（構造化データ抽出時）
- 対象サイトの利用規約確認（爬取が許可されているか）

## 根拠

> 给一个网址，自动爬完整站，返回HTML/Markdown/JSON

> 支持 JS 渲染（React/Vue 动态站也能爬）

> 增量爬取：只爬更新过的页面，省时省钱

> 要什么数据，直接返回结构化 JSON
