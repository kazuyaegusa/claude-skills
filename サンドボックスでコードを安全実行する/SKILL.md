# サンドボックスでコードを安全実行する

> エージェントが生成したコードをホスト環境から隔離されたサンドボックス内で実行し、結果をエージェントにフィードバックする

- 出典: https://x.com/dify_base/status/2031190239885353023
- 投稿者: AX Base| AX情報発信メディア
- カテゴリ: agent-orchestration

## なぜ使うのか

LLMが生成したコードはホスト環境に任意の副作用を与えうるため、隔離実行によってシステムへの意図しない変更やセキュリティリスクを防ぐ

## いつ使うのか

エージェントが生成・実行するコードの内容を事前に保証できないとき、または本番環境でエージェントを動かすとき

### 具体的な適用場面

- 複数ステップの調査・コード生成・検証を含む自律ワークフローをPythonで構築したいとき
- Claude Codeを既存のマルチエージェントパイプラインに組み込んでコーディングタスクを委譲したいとき
- エージェントが生成したコードをホスト環境に影響させず安全に実行したいとき

## やり方

1. DeerFlowのコーダーサブエージェントがコードを生成する
2. フレームワーク内蔵のサンドボックス実行エンジン（Dockerコンテナまたはプロセス分離）がコードを受け取る
3. 実行結果（stdout/stderr/returncode）をエージェントに返却し、次のステップへ反映
※具体的なサンドボックス設定はconf.yamlの`sandbox`セクションで制御（詳細はリポのREADME参照）

### 入力

- エージェント生成コード（Python等）

### 出力

- 実行結果（stdout/stderr）
- エラー情報

## 使うツール・ライブラリ

- DeerFlow (bytedance/deer-flow)
- Docker（サンドボックスバックエンドとして）

## 前提知識

- Python 3.10以上の実行環境
- git によるリポジトリクローン操作の知識
- LLMプロバイダーのAPIキーまたはClaude Code subscription認証
- Dockerの基礎知識（サンドボックス実行を使う場合）
