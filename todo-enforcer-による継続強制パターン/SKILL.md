# Todo Enforcer による継続強制パターン

> LLMエージェントがタスクを途中で止めた場合、自動的に再開させて完了まで強制継続する仕組み

- 出典: https://github.com/code-yeongyu/oh-my-openagent
- 投稿者: code-yeongyu
- カテゴリ: agent-orchestration

## なぜ使うのか

LLMは複雑なタスクで途中で諦めたり、「残りはユーザーがやってください」と逃げることがある。これを防ぐため、TODOリストを監視し、未完了があれば継続を強制する

## いつ使うのか

複雑な多段階タスク、LLMが途中で止まる問題がある場合、完全自動化したい場合

## やり方

1. エージェントがタスクを開始すると、内部的にTODOリストを生成
2. Todo Enforcerがリストを監視
3. エージェントが途中で止まったり、未完了項目があれば自動的に「続きをやれ」と指示
4. Sisyphusは「石を転がし続ける」ように、完了まで実行

### 入力

- タスクのTODOリスト（自動生成）

### 出力

- 途中停止しない実行
- タスク完了保証

## 使うツール・ライブラリ

- oh-my-opencode Todo Enforcer hook

## 前提知識

- OpenCodeの基本的な使い方
- LLM API（Claude, GPT, Gemini等）のアクセス権
- npm/Node.js環境
- マルチエージェントオーケストレーションの概念理解
- LSP/ASTの基本的な理解（リファクタリング機能を使う場合）
