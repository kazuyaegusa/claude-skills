# AgentShieldでセキュリティスキャンを実行する

> 102のセキュリティルール・1,282テストケースをプロジェクトコードに対して自動実行し、脆弱性レポートを生成する

- 出典: https://x.com/l_go_mrk/status/2037489559186145787
- 投稿者: AI駆動塾
- カテゴリ: claude-code-workflow

## なぜ使うのか

手動のコードレビューでは見落としやすいOWASP Top 10・インジェクション・認証不備等をパターンマッチングで網羅的に検出でき、セキュリティ専門知識がなくても一定水準の検証が行える

## いつ使うのか

PRマージ前、本番デプロイ前、または外部公開するOSSリポジトリのセキュリティ検証をしたいとき

### 具体的な適用場面

- 新規プロジェクト立ち上げ時に、Claude Codeの能力を最大化した状態でスタートしたいとき
- 既存プロジェクトに対して脆弱性チェックを自動化したいとき（OWASP等102ルール）
- フロントエンド・バックエンド・テスト・セキュリティ等を同時進行で進めたいマルチドメインタスクのとき

## やり方

1. `npx ecc-agentshield` でAgentShieldのみを単体導入（ECCフル導入済みならスキップ）
2. Claude Codeのチャットで `/security-scan` または `/vibecheck` を入力
3. スキャン対象ディレクトリを指定（デフォルトはカレント）
4. 102ルールによる自動解析が走り、脆弱性の種類・深刻度・該当ファイル行番号がレポートとして出力される

### 入力

- スキャン対象のソースコードディレクトリ
- ecc-agentshield導入済みのClaude Code環境

### 出力

- 脆弱性レポート（種類・深刻度・ファイルパス・行番号）
- 修正提案（自動修正可能なものはパッチ提示）

## 使うツール・ライブラリ

- ecc-agentshield（npm package）
- Claude Code CLI

## コード例

```
npx ecc-agentshield
# 導入後はClaude Codeチャットで:
# /security-scan
# または /vibecheck
```

## 前提知識

- Claude Code CLIのインストール（npm install -g @anthropic-ai/claude-code または brew install claude）
- Node.js v18以上（npx実行のため）
- Claude ProまたはMax サブスクリプション（サブエージェント並列実行にはトークン消費が多い）
