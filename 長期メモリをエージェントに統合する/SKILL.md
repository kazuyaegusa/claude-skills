# 長期メモリをエージェントに統合する

> セッションをまたいで過去の調査結果・判断・コンテキストを保持し、後続タスクで再利用できるようにする

- 出典: https://x.com/dify_base/status/2031190239885353023
- 投稿者: AX Base| AX情報発信メディア
- カテゴリ: agent-orchestration

## なぜ使うのか

LLMのコンテキストウィンドウはセッション内に限られるため、長期記憶を外部ストアに永続化することで、繰り返しの調査や関連タスクで既知情報を再活用できる

## いつ使うのか

同一ドメインの調査タスクを繰り返し実行する場合、または長時間・多ステップのプロジェクトでエージェントの記憶を継続させたいとき

### 具体的な適用場面

- 複数ステップの調査・コード生成・検証を含む自律ワークフローをPythonで構築したいとき
- Claude Codeを既存のマルチエージェントパイプラインに組み込んでコーディングタスクを委譲したいとき
- エージェントが生成したコードをホスト環境に影響させず安全に実行したいとき

## やり方

1. DeerFlowの長期メモリモジュールを有効化（conf.yamlで`memory: enabled: true`等を設定）
2. エージェントが処理した結果が自動でベクトルDBまたはKVストアに保存される
3. 次回タスク実行時、関連記憶が自動取得されてコンテキストに注入される
※ストアのバックエンド（SQLite/Redisなど）はconf.yamlで切り替え可能

### 入力

- タスク処理結果
- conf.yamlのメモリバックエンド設定

### 出力

- 永続化された記憶エントリ
- 次タスクへの自動コンテキスト注入

## 使うツール・ライブラリ

- DeerFlow (bytedance/deer-flow)
- SQLite / Redis（メモリバックエンド）

## 前提知識

- Python 3.10以上の実行環境
- git によるリポジトリクローン操作の知識
- LLMプロバイダーのAPIキーまたはClaude Code subscription認証
- Dockerの基礎知識（サンドボックス実行を使う場合）
