# サブエージェント駆動開発でタスクを並列実行する

> 実装計画をYAGNI原則に従って最小サブタスクに分割し、各タスクを独立したサブエージェントに割り当ててTDD（赤→緑）で並列実行させる。

- 出典: https://x.com/jiroucaigou/status/2035964407201849564
- 投稿者: 努力赚钱的菜狗
- カテゴリ: agent-orchestration

## なぜ使うのか

単一エージェントの長時間実行より小タスク×複数エージェントの方が問題の局所化が容易で、1つのサブタスク失敗が他に波及しない。並列化でトータルの実装時間も短縮される。

## いつ使うのか

実装計画の中に独立した複数のタスクが存在するとき（例: 認証実装とDB設計が並列可能な場合）

### 具体的な適用場面

- AIが生成したコードは動くが、プロジェクト拡大後に保守不能になるパターンが繰り返されているとき
- 要件が曖昧なままAIに着手させて後から仕様ズレが発覚するケースを防ぎたいとき
- 複数の独立機能を並列開発してAIの作業スループットを上げたいとき
- AIエージェントに数時間の自律作業をさせたいが品質保証が不安なとき

## やり方

1. `superpowers:execute-plan` スキルが自動的に計画をサブタスクに分割する。2. 各サブタスクをClaude CodeのAgentツールで別インスタンスとして起動。3. 各サブエージェントはTDD（テスト先書き→実装→グリーン確認）で作業。4. 全サブエージェントの完了を待ってからレビューフェーズへ。手動でやる場合はClaude Codeの `Agent` ツール呼び出しで `isolation: 'worktree'` を指定してサブエージェントを並列起動する。

### 入力

- 承認済みの実装計画（タスクリスト）
- 各タスクの依存関係マップ

### 出力

- 各タスクの実装コード
- テストスイート（全グリーン）

## 使うツール・ライブラリ

- Superpowers
- Claude Code Agent tool
- superpowers:subagent-driven-development スキル
- superpowers:dispatching-parallel-agents スキル

## コード例

```
# Claude CodeのAgentツールでサブエージェントを並列起動する例
# isolation: 'worktree' で物理隔離も同時に有効化
Agent({
  subagent_type: 'general-purpose',
  isolation: 'worktree',
  prompt: 'task_a の実装をTDDで行う',
  run_in_background: true
})
```

## 前提知識

- Claude Code CLI または Cursor がインストール済みであること
- Gitの基本操作（branch, merge, worktree）を理解していること
- TDD（テスト駆動開発）の赤→緑サイクルの概念を知っていること

## 根拠

> 「1.先疯狂提问，把你的需求完全理解，你同意后并生成标准设计文档」—— 要件深掘りと設計文書生成の強制

> 「2.开新分支+隔离工作区，确保不会搞乱主代码」—— ブランチ+隔離ワークスペースの使用

> 「3.把任务拆分成最小子任务」—— 最小サブタスク分割

> 「4.交给子Agent执行并强制测试，最小代码也要通过测试」—— サブエージェントへの委任とテスト強制

> 「5.对完成的任务进行双重审查，不仅要符合设计规格，更要检查代码质量，有一项不过关就不放行」—— 仕様適合＋コード品質のダブルレビュー
