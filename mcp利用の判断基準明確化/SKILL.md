# MCP利用の判断基準明確化

> 持続的ブラウザコンテキスト・豊富な内省・反復推論が必要なユースケースでのみMCPを選択

- 出典: https://github.com/microsoft/playwright-mcp
- 投稿者: microsoft
- カテゴリ: context-management

## なぜ使うのか

MCPの強みは永続状態管理とページ構造への反復的推論だが、これはトークンコストとトレードオフ。高スループットなコーディングタスクでは過剰

## いつ使うのか

ブラウザ自動化の統合方式を選択する設計段階

## やり方

1. タスクが「探索的自動化」「自己修復テスト」「長時間自律ワークフロー」に該当するか判定
2. 該当する場合、持続的ブラウザコンテキストがトークンコストを上回る価値を生むか評価
3. YesならMCP、NoならCLI+SKILLSを選択

### 入力

- タスク要件（スループット vs 内省深度）
- コンテキストウィンドウ制約

### 出力

- 最適な統合方式の選択

## 前提知識

- MCP（Model Context Protocol）の基本概念
- コーディングエージェントのコンテキストウィンドウ制約
- Playwrightの基本操作知識
- トークン効率とエージェント性能の関係
