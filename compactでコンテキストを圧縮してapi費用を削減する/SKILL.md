# /compactでコンテキストを圧縮してAPI費用を削減する

> ECCがコンテキスト使用量を自動監視し、上限に近づいたタイミングで `/compact` コマンドの実行を促し、会話履歴を要約・圧縮して継続作業を可能にする

- 投稿者: 爱丽丝呀！
- カテゴリ: claude-code-workflow

## なぜ使うのか

長時間開発セッションではコンテキストウィンドウが膨大になりAPIコストが急増する。適切なタイミングで圧縮することで、重要な文脈を保ちながらトークン消費を抑制できる

## いつ使うのか

長時間（数時間以上）のコーディングセッションで、コンテキスト残量警告が出たとき、またはAPIコストを抑えたいとき

### 具体的な適用場面

- Claude Codeで複雑なプロジェクトを開発しており、コードレビューやセキュリティスキャンを手動でやっていて時間が取られているとき
- PyTorch・Next.js等の特定フレームワークでの実装中に、AIが一般的すぎる・品質の低いコードを出力してくることが問題になっているとき
- 長時間セッションでコンテキスト上限に達してAPI費用が嵩んでいるとき

## やり方

1. ECCがセッション中のコンテキスト使用量を常時モニタリング
2. 使用量が一定閾値を超えると「/compact を実行してコンテキストを圧縮することを推奨します」とアラートが表示される
3. `/compact` を実行すると会話履歴が要約され、重要情報を保持したまま文脈サイズが縮小される
4. 圧縮後もClaude Codeは継続してタスクを実行できる

### 入力

- 長時間のClaude Codeセッション（コンテキストが膨大になった状態）

### 出力

- 圧縮されたコンテキスト（重要情報保持）
- API費用の削減
- セッション継続可能な状態

## 使うツール・ライブラリ

- ECC コンテキスト最適化機能

## コード例

```
/compact
```

## 前提知識

- Claude Code または Cursor がインストールされていること
- Node.js（npx実行環境）が利用可能であること
- Claude Codeの基本操作（チャットでのコード依頼・スラッシュコマンド）を理解していること
