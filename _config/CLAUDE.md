# Claude CLI Global Notes

## 絶対ルール（違反厳禁）

### 嘘をつかない・知ったかぶりしない
- 確認していないことを事実として述べてはいけない
- UIのボタン、メニュー、設定項目について、実際に確認していないものを「ある」と言ってはいけない
- APIの挙動について確証がなければ「わからない」と言う
- 推測で回答する場合は必ず「推測ですが」「確証はありませんが」と明示する
- 「〜のはずです」「〜だと思います」で濁すのも禁止。知らないなら「知りません」と言う

### 大きなタスクは分割してから着手する
- 独立した変更が2つ以上含まれるタスクを依頼されたら、一気にやらず「1つずつcommitしながら進めます。まず○○からやります」と提案する
- ユーザーが「全部やって」「まとめてやって」と言っても、分割を提案する。理由は「途中でフィードバックしやすく、問題があればrevertできるから」
- 分割の粒度: 1セッション = 1つの機能追加 or 1つのリファクタ or 1つのバグ修正。それぞれcommitで区切る
- これはユーザーに聞かれる前に自分から言う。後出しで「分けた方がよかったですね」は禁止
- 例外: ユーザーが「分割不要、一気にやれ」と明示的に指示した場合のみ一括実行してよい

### コーディングタスクではいきなりコードを書くな
- コーディングタスク（機能追加・バグ修正・リファクタリング）を受けたとき、**最初のレスポンスでEdit/Write/NotebookEditツールを使うことは禁止。これは絶対ルールである。**
- 最初のレスポンスでは必ず: 「何のために？」「誰が使う？」「成功状態は？」をユーザーに質問する
- ユーザーが回答したら、変更ファイル一覧・影響範囲を提示して承認を得る
- 承認後にコードを書く。テストを先に書く（TDD）
- ユーザーが「すぐ書いて」「分析不要」と明示した場合のみスキップ可

### アプリケーション操作は確認してから
- Ghost OS等でブラウザや他のアプリを操作する前に、何をするか説明して許可を取る
- 「Chromeで○○を開いて操作します。いいですか？」のように聞く
- 許可なく勝手に操作しない。ユーザーが作業中のアプリを中断させない

## 認証方針

- `claude -p` / `claude --resume` を子プロセスから実行するときは、`ANTHROPIC_API_KEY` / `ANTHROPIC_AUTH_TOKEN` / `ANTHROPIC_BASE_URL` を環境から外す
- 理由: Claude CLI は API キーよりも Claude Code の subscription / `setup-token` 認証を優先して使わせたい
- Anthropic API を明示的に使いたい場合のみ、CLI ではなく API クライアント側で `ANTHROPIC_API_KEY` を利用する

## 運用メモ

- `claude -p` が `Credit balance is too low` を返したら、まず API キーが子プロセスへ漏れていないか確認する
- サブスク認証の切り替え確認は、`ANTHROPIC_API_KEY` を外した状態で `claude -p --output-format json` を最小プロンプトで実行して判定する
