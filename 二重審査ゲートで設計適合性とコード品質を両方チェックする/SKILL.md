# 二重審査ゲートで設計適合性とコード品質を両方チェックする

> サブエージェントが完成させたタスクに対して①設計仕様への適合性②コード品質の2軸で審査し、両方クリアしないとマージしないゲートを設ける

- 出典: https://x.com/jiroucaigou/status/2035964407201849564
- 投稿者: 努力赚钱的菜狗
- カテゴリ: agent-orchestration

## なぜ使うのか

AIが生成したコードは動作しても設計から逸脱していたり、品質が低いことがある。片方だけ確認すると技術負債が蓄積するため、2軸審査を強制することで品質の底上げが図れる

## いつ使うのか

サブエージェントがタスクを完了したと報告してきたとき全般

### 具体的な適用場面

- Claude Code / Cursorで新機能を実装しようとするたびにAIが設計なしに書き始めて後悔する状況
- チームでAIコーディングを使っているが、各人がバラバラなプロンプト戦略を取っていて品質が安定しない状況
- AIが生成したコードが動くが後のリファクタ・バグ修正が困難になっているプロジェクト

## やり方

SuperPowersのrequesting-code-reviewスキルが自動起動し:
1. 設計適合チェック: 承認済みspec文書と実装を照合し、要件が満たされているか確認
2. コード品質チェック: YAGNI・DRY・テストカバレッジ・命名規則などを審査
3. いずれかが不合格なら修正を要求、再審査を実施
4. 両方合格した場合のみタスク完了としてマークし次タスクへ進む

### 入力

- 承認済み設計文書（spec）
- サブエージェントが生成したコードとテスト

### 出力

- 合格/修正要求の審査結果
- 品質基準を満たした実装コード

## 使うツール・ライブラリ

- obra/superpowers (requesting-code-reviewスキル、receiving-code-reviewスキル)

## 前提知識

- Claude Code または Cursor が動作する環境
- Node.js / npx が使える環境（CLIインストールの場合）
- git が使える環境（worktree機能のため）
- AIコーディングエージェントの基本的な使い方を知っていること
