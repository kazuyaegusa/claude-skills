# プランモードを常に最初に使う

> 実装に入る前に必ずPlan Mode（`Shift+Tab` またはメニューから）でClaudeに計画を立てさせ、承認してから実行に移る

- 出典: https://x.com/labelmake/status/2032652943250338098
- 投稿者: Kyohei - OSS, 外資IT
- カテゴリ: claude-code-workflow

## なぜ使うのか

計画なしに実装を始めると方向性のズレが後で判明し、大きなrevertが必要になる。計画段階でのフィードバックが最もコスト効率が良い

## いつ使うのか

新機能開発、大きなリファクタリング、複数ファイルにまたがる変更など、影響範囲が広いタスクの開始時

## やり方

1. Claude Codeを起動しPlan Modeを有効化
2. タスクを説明し、Claudeに実装計画を立てさせる
3. 計画を読んで問題がなければ承認
4. 必要であれば別のClaudeインスタンス（またはCodex等の別モデル）に計画をレビューさせる
5. 承認後に通常モードで実装を実行

### 入力

- タスク要件・仕様

### 出力

- 承認済み実装計画
- フェーズ別のゲート付き計画（各フェーズにunit/automation/integrationテスト含む）

## 使うツール・ライブラリ

- Claude Code Plan Mode
- /plan
