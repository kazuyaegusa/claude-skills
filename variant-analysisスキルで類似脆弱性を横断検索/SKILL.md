# variant-analysisスキルで類似脆弱性を横断検索

> 1つの脆弱性パターンを見つけたら、コードベース全体から同様のパターンを検出する

- 出典: https://github.com/trailofbits/skills
- 投稿者: trailofbits
- カテゴリ: claude-code-workflow

## なぜ使うのか

脆弱性は孤立して存在せず、同じコーディングミスが複数箇所に存在することが多い。1つ見つけたらバリアント（類似例）を網羅的に検出することで、修正漏れを防ぐ

## いつ使うのか

1つのセキュリティバグを発見した後、同じ種類の問題が他にも存在しないか確認したい時

## やり方

1. variant-analysisスキルをインストール
2. 最初に見つけた脆弱性パターンをスキルに入力
3. スキルがパターンマッチング戦略を策定
4. コードベース全体をスキャンして類似箇所をリストアップ
5. 検出結果を精査し、実際の脆弱性か判定

### 入力

- 既知の脆弱性パターン（コード例またはSARIF/Semgrep結果）
- 検索対象のコードベース

### 出力

- 類似パターンが存在するファイルと行番号のリスト
- 各候補の脆弱性評価

## 使うツール・ライブラリ

- variant-analysis スキル
- Semgrep/CodeQL（バックエンド）

## 前提知識

- Claude CodeまたはCodexのインストールと基本的な使い方
- セキュリティ監査・脆弱性分析の基礎知識
- 静的解析ツール（Semgrep, CodeQL等）の概要理解
- スマートコントラクト監査の場合は対象ブロックチェーンの知識
- 暗号解析スキルを使う場合はサイドチャネル攻撃の基礎知識
