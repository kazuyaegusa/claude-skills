# /reviewで本番品質のコードレビューを自動実行する

> コードを書いた後にClaude Code上で `/review` コマンドを打つと、ReviewerエージェントとSecurity Scanエージェントが連携して本番レベルの品質審査・脆弱性スキャンを行う

- 出典: https://x.com/btcqzy1/status/2038853050643980475
- 投稿者: 爱丽丝呀！
- カテゴリ: claude-code-workflow

## なぜ使うのか

コードを書いた本人がセルフレビューしても見落としが多く、外部レビュアーを都度呼ぶのは非現実的。本番基準で自動審査することで、低品質コードやセキュリティホールをコミット前に排除できる

## いつ使うのか

実装が一区切りついたとき、PRを出す前、セキュリティが重要な機能（認証・決済・API公開）を実装した直後

### 具体的な適用場面

- Claude Codeで複雑なプロジェクトを開発しており、コードレビューやセキュリティスキャンを手動でやっていて時間が取られているとき
- PyTorch・Next.js等の特定フレームワークでの実装中に、AIが一般的すぎる・品質の低いコードを出力してくることが問題になっているとき
- 長時間セッションでコンテキスト上限に達してAPI費用が嵩んでいるとき

## やり方

1. ECC導入済みのClaude Codeでコードを書く
2. コーディングが一区切りついたら、チャット欄に `/review` と入力して送信
3. ReviewerエージェントがコードのBug・設計問題・ベストプラクティス違反を列挙
4. Security ScanエージェントがOWASPや既知脆弱性パターンを照合してセキュリティリスクを報告
5. 指摘事項を確認してClaude Codeに修正を依頼

### 入力

- レビュー対象のコード（Claude Codeのコンテキスト内に存在すること）

### 出力

- 本番品質基準での指摘リスト（Bug・設計問題・ベストプラクティス違反）
- セキュリティ脆弱性スキャン結果

## 使うツール・ライブラリ

- ECC Reviewer Agent
- ECC Security Scan Agent

## コード例

```
/review
```
