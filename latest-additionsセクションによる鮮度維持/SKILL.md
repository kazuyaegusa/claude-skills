# "Latest Additions"セクションによる鮮度維持

> README冒頭に最新追加リソースを3-4個ハイライト表示し、各リソースに詳細な説明を付ける

- 出典: https://github.com/hesreallyhim/awesome-claude-code
- 投稿者: hesreallyhim
- カテゴリ: claude-code-workflow

## なぜ使うのか

定期的な更新があることを示すことで、リポジトリが活発にメンテナンスされていることを証明し、訪問者の信頼を獲得する。また、リピーターが新規追加を素早く発見できる

## いつ使うのか

継続的に更新されるリソース集で、新規追加を強調したい場合

## やり方

1. 新規リソース追加時に"Latest Additions"セクションを更新
2. 各項目に作者、リンク、2-4文の説明を記載
3. 古い項目は通常のカテゴリーセクションに移動
4. 3-4項目を維持し、最新性を保つ

### 入力

- 新規追加リソース情報
- タイムスタンプまたは追加順序

### 出力

- "Latest Additions"セクション（3-4項目）
- 通常カテゴリーへの移行済みリスト

## 使うツール・ライブラリ

- Markdown

## コード例

```
## Latest Additions

- [claude-devtools](https://github.com/matt1398/claude-devtools) by [matt1398](https://github.com/matt1398) - A well-designed desktop app that provides detailed observability into your Claude Code sessions...
```

## 前提知識

- Claude Codeの基本的な使い方（CLI、設定ファイル、コマンド実行）
- GitHubの基本操作（リポジトリのクローン、READMEの閲覧）
- Markdownの読解
- AI開発ツールの一般的な概念（エージェント、フック、プロンプト等）

## 根拠

> "Latest Additions" セクションに claude-devtools, agnix, Codebase to Course, Ruflo を掲載
