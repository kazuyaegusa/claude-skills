# Collections セクションによるメタカタログ整理

> 複数のスキルをバンドルしたコレクション（例: OpenPaw, Agent Almanac）を別セクションにまとめ、エコシステム全体を俯瞰できるようにする

- 出典: https://github.com/BehiSecc/awesome-claude-skills
- 投稿者: BehiSecc
- カテゴリ: claude-code-workflow

## なぜ使うのか

個別スキルとスキル集を同列に並べると、ユーザーが「どちらから始めるべきか」を判断できない

## いつ使うのか

エコシステム内にスキルバンドルやマーケットプレイスが登場した時

## やり方

1. Collections セクション（🗂️）を作成
2. スキル集のリポジトリ、含まれるスキル数、対象領域を明記
3. 個別スキルセクションとは分離して配置

### 入力

- スキルコレクションのリポジトリ情報
- 含まれるスキル数・対象領域

### 出力

- Collections セクション
- 各コレクションの概要

## コード例

```
## 🗂️ Collections
- [OpenPaw](https://github.com/daxaur/openpaw) - 38-skill bundle that turns Claude Code into a personal assistant.
- [Agent Almanac](https://github.com/pjt222/agent-almanac) - 317 skills, 65 agents, and 14 teams for Claude Code
```

## 前提知識

- Claude Code の基本的な使い方（スキルのインストール方法）
- GitHub リポジトリの構造と README.md の役割
- Markdown 記法
- awesome-* リストの概念（例: awesome-python, awesome-go）
