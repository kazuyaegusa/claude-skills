# 完了タスクに仕様適合＋コード品質のダブルレビューを強制する

> サブエージェントが完了報告をしたタスクに対し、「設計仕様への適合」と「コード品質」を独立した2軸で審査し、どちらか一方でも不合格なら差し戻す。

- 出典: https://x.com/jiroucaigou/status/2035964407201849564
- 投稿者: 努力赚钱的菜狗
- カテゴリ: agent-orchestration

## なぜ使うのか

仕様通りでも品質が低いコードは技術的負債になり、品質が高くても仕様から外れたコードは後でバグになる。2軸を独立評価することで両方の問題を同時に防げる。

## いつ使うのか

サブエージェントが「タスク完了」と報告したとき、PRをmainにマージする前の最終確認時

### 具体的な適用場面

- AIが生成したコードは動くが、プロジェクト拡大後に保守不能になるパターンが繰り返されているとき
- 要件が曖昧なままAIに着手させて後から仕様ズレが発覚するケースを防ぎたいとき
- 複数の独立機能を並列開発してAIの作業スループットを上げたいとき
- AIエージェントに数時間の自律作業をさせたいが品質保証が不安なとき

## やり方

1. `superpowers:requesting-code-review` スキルが自動トリガーされ、設計文書と実装を突き合わせて仕様適合チェックを実行。2. `superpowers:verification-before-completion` スキルでコード品質（TDD準拠、YAGNI違反、DRY違反等）をチェック。3. 両方パスで初めて次のタスクへ進む。4. 不合格の場合はエージェントが差し戻し理由を明示して再実装を要求。

### 入力

- サブエージェントの実装コード
- 承認済み設計文書（SPEC）
- テスト結果

### 出力

- 合格/差し戻しの判定
- 差し戻しの場合は具体的な修正指摘

## 使うツール・ライブラリ

- Superpowers
- superpowers:requesting-code-review スキル
- superpowers:receiving-code-review スキル
- superpowers:verification-before-completion スキル

## 前提知識

- Claude Code CLI または Cursor がインストール済みであること
- Gitの基本操作（branch, merge, worktree）を理解していること
- TDD（テスト駆動開発）の赤→緑サイクルの概念を知っていること

## 根拠

> 「1.先疯狂提问，把你的需求完全理解，你同意后并生成标准设计文档」—— 要件深掘りと設計文書生成の強制

> 「2.开新分支+隔离工作区，确保不会搞乱主代码」—— ブランチ+隔離ワークスペースの使用

> 「3.把任务拆分成最小子任务」—— 最小サブタスク分割

> 「4.交给子Agent执行并强制测试，最小代码也要通过测试」—— サブエージェントへの委任とテスト強制

> 「5.对完成的任务进行双重审查，不仅要符合设计规格，更要检查代码质量，有一项不过关就不放行」—— 仕様適合＋コード品質のダブルレビュー
