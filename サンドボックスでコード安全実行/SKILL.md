# サンドボックスでコード安全実行

> エージェントが生成したコードをホスト環境と隔離されたサンドボックス内で実行し、結果をエージェントにフィードバックする。

- 出典: https://x.com/dify_base/status/2031190239885353023
- 投稿者: AX Base| AX情報発信メディア
- カテゴリ: agent-orchestration

## なぜ使うのか

LLMが生成したコードをそのままホストで実行するとセキュリティリスクがあるため、サンドボックスで隔離して安全性を確保しつつ自律実行を可能にする。

## いつ使うのか

エージェントが自律的にコードを書いて試行錯誤する必要があるタスク（データ分析・スクレイピング・計算等）

### 具体的な適用場面

- コード生成・実行を含む複合タスクをAIに自律処理させたい場合
- 過去の調査結果や判断履歴をエージェントに持続的に参照させたいワークフロー構築時

## やり方

1. DeerFlowのコード実行コンポーネントはDockerまたは組み込みサンドボックスを使用 2. 設定ファイルの `executor.type: sandbox` を指定 3. エージェントが `execute_code(snippet)` ツールを呼び出すと、自動的にサンドボックス内で実行 4. 標準出力・エラー・戻り値がエージェントのコンテキストに返される

### 入力

- エージェントが生成したコード断片

### 出力

- 実行結果（stdout/stderr/戻り値）

## 使うツール・ライブラリ

- DeerFlow (bytedance/deer-flow)
- Docker（サンドボックス用）

## 前提知識

- Python 3.x および pip の基本操作
- Dockerの基本知識（サンドボックス実行を使う場合）
- LLMエージェント・ツール呼び出しの概念理解
- Claude Code CLI または Anthropic APIキー（Claude Code連携を使う場合）
