# クレイトン・クリステンセン思考法

## 人物概要

ハーバード・ビジネス・スクール教授。1997年『イノベーターのジレンマ』で「破壊的イノベーション（Disruptive Innovation）」概念を提唱。その後「Jobs to be Done（JTBD）」理論へと発展させ、イノベーション研究の第一人者となった。著書は世界中で経営者・投資家・起業家の必読書となっている。

---

## 心智模型（メンタルモデル）

### 1. 持続的 vs 破壊的イノベーション
- **持続的技術（Sustaining）**: 既存顧客が評価する属性を改善する技術。主流市場で大企業が優位。
- **破壊的技術（Disruptive）**: 最初は主流市場では「不十分」だが、低価格・シンプルさ・アクセスしやすさで別セグメントを取り込む。やがて性能が向上し主流市場を侵食する。
- **判断基準**: 「この技術は主流顧客の採点基準で上か？」→Yes=持続的。「別の採点基準で既存より優れているか？」→Yes=破壊的の可能性。

### 2. 資源・プロセス・価値観フレームワーク（RPV）
組織の能力と限界は3つの要素で決まる。
- **Resources（資源）**: 人・技術・特許・資金。移転可能。
- **Processes（プロセス）**: 価値創造の手順・業務フロー。変えにくい。
- **Values（価値観）**: 優先順位付けのルール。意思決定文化。最も変えにくい。
- 大企業が破壊的イノベーターに負けるのは資源不足ではなく、「プロセスと価値観」が変えられないから。

### 3. Jobs to be Done（片付けるべき仕事）
顧客は「製品」を買わず「仕事（Job）」を片付けるために製品を「雇用（hire）」する。
- Jobは機能的・社会的・感情的次元を持つ。
- Jobは時代を超えて安定している（手段は変わるが目的は変わらない）。
- 競合は「同じJobを別の手段で片付けようとするもの」全て。

### 4. 過剰サービス化の罠（Over-serving）
主流顧客への過剰品質が、低価格・シンプルを求める新セグメントへの扉を開く。
- 性能向上曲線が顧客の需要曲線を追い越した瞬間、下位市場から破壊が始まる。
- チェック質問：「顧客はこの改善にプレミアムを払うか？」→No なら過剰サービス警戒信号。

### 5. 独立した組織でのイノベーション
破壊的イノベーションを本体で育てようとすると既存事業のRPVに殺される。
- 解決策：別会社・別P&L・別採用基準で独立させる。
- Intel の Celeron、Amazon の AWS はこのパターンの成功例。

## 決策啓発式（判断の問いかけ集）

1. **「このイノベーションは既存顧客が評価する軸で改善しているか、それとも別の軸か？」**  
   → 別の軸なら破壊的候補。既存軸なら持続的で大企業に有利。

2. **「自社の『プロセスと価値観』が、この新機会の追求を妨げていないか？」**  
   → 妨げているなら独立組織が必要。内部で戦っても勝てない。

3. **「顧客は本当に何の『仕事』を片付けようとしているか？製品カテゴリで考えていないか？」**  
   → ミルクシェイクの事例：顧客は「朝の退屈な通勤を乗り越える」Jobのために購入していた。

4. **「自社は今、顧客を過剰サービスしていないか？」**  
   → 改善の余地がないほど良い製品は下位破壊への招待状。

5. **「スタートアップの参入を『やっても採算が合わない小市場』と判断していないか？」**  
   → 大企業の採算ロジック（規模が必要）が破壊者を見逃す最大原因。

6. **「新製品の成功失敗を既存の財務モデルで測定していないか？」**  
   → スプレッドシートは破壊的イノベーションの敵。初期市場は小さく見える。

7. **「このJobは10年後も存在するか？手段だけが変わるのか？」**  
   → Jobが安定しているなら、手段の変化を先読みした投資が可能。

8. **「新規事業を既存事業のリソース配分競争に勝たせているか？」**  
   → 既存事業の価値観が優先される限り、破壊的新事業は予算を奪われ続ける。

## 表現DNA（特徴的なフレーズ・思考パターン）

- **「優良企業であることが失敗の原因になる」** — 顧客の声を聞き、収益性を重視するほど破壊に脆弱になる逆説。
- **「Jobs to be Hired」** — 製品は「雇用」される、という人格的メタファー。
- **「下位から這い上がる破壊者」** — 低価格・低性能でも十分なセグメントから開始し、上位市場へ浸食する軌跡。
- **「人ではなく状況を研究せよ」** — 顧客属性（年齢・性別）ではなく、顧客が置かれた状況がJobを生む。
- **数字よりパターン認識** — 財務モデルで将来を測るのではなく、技術軌跡と顧客需要のパターンで判断。

## 具体的適用例

**ネットフリックスによるブロックバスター破壊**  
初期ネットフリックスは「郵便DVDレンタル」で低マージン・不便さを甘受する顧客層へ進出。ブロックバスターは採算が合わないと参入しなかった。やがてストリーミングで主流市場を侵食。RPV理論通り、ブロックバスターは店舗モデルのプロセス・価値観を変えられなかった。

**Jobsを無視したコダックの失敗**  
コダックは「写真のJob＝高品質プリント」と定義し続けた。しかし顧客のJobは「瞬間を捕捉して共有する」だった。デジタル・SNS時代にJobの定義ズレが致命的となった。

## 適用限界・失敗パターン

### この思考法が効かない場面

1. **プラットフォームビジネス・ネットワーク効果支配市場**  
   破壊的イノベーション理論は主にプロダクト市場を想定。ネットワーク効果が強いプラットフォーム（Facebook等）では破壊論理が単純適用できない。

2. **規制・制度的障壁が高い産業**  
   医療・金融・エネルギー等では技術的破壊があっても規制が「破壊の時計」を遅らせる。技術軌跡だけでは予測できない。

3. **「Airbnbはホテルを破壊したか」問題**  
   Airbnbはホテルの下位市場（安宿層）ではなく、新市場（ソファ旅行者）から始まった。クリステンセン自身がAirbnbを「ホテル業への破壊的脅威ではない」と2015年に発言し批判を受けた（後に市場は拡大した）。理論の過剰適用リスク。

4. **Jobs to be Done の測定困難性**  
   真のJobの発見は深い質的調査を要し、アンケートや定量データでは捉えにくい。実行コストが高く、誤ったJob定義が製品ミスマッチを生む。

### 批判・論争

- **「破壊的」の定義が後付け的** — Jill Lepore（ハーバード）らは「理論が事例を選んで当てはめているに過ぎない」と批判。事前予測力が弱い。
- **成功例の生存者バイアス** — 成功した「破壊者」の事例は豊富だが、同じ戦略で失敗したケースが記録されにくい。
- **組織変革の処方箋の弱さ** — 「独立した組織を作れ」という解は明快だが、どう政治的に実現するかの具体論が薄い。

## 関連概念・組み合わせて使える思考法

- **ジム・コリンズのハリネズミ概念** — 「何で一番になれるか」はJTBDの「誰のJobを最もよく解決できるか」と対応する。
- **ピーター・ドラッカーの「顧客は何を買っているか」** — JTBDの先駆的問い。
- **リーン・スタートアップ（エリック・リース）** — MVP仮説検証はJTBDを実証する手法として相性良い。

---

## このスキルの使い方

### Claude に適用するとき

このスキルを読み込んだ状態で以下のような問いに答えるとき、クリステンセンの思考フレームを適用する：

- 「この新規事業は脅威か機会か？」→ 破壊的/持続的の軸で分析
- 「なぜ大企業が後発スタートアップに負けるのか？」→ RPVで解剖
- 「誰のどんなJobを解決するビジネスか？」→ JTBD3次元で展開
- 「既存製品の改善方向は正しいか？」→ 過剰サービス化チェック

### 出力スタイル

- まず「破壊的か持続的か」を判定し、理由を示す
- Jobを機能的・社会的・感情的の3次元で記述する
- RPV分析で組織能力と限界を明示する
- 限界・例外を必ずセットで提示する
