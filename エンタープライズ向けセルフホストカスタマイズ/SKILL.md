# エンタープライズ向けセルフホスト＋カスタマイズ

> Activepiecesをセルフホスト（ネットワークギャップ環境含む）で運用し、ブランディングからアクセス制御まで完全にカスタマイズすることで、エンタープライズ要件に対応する

- 出典: https://github.com/activepieces/activepieces
- 投稿者: activepieces
- カテゴリ: automation-pipeline

## なぜ使うのか

データ主権、セキュリティ、コンプライアンス要件により、外部SaaSを利用できない企業が多い。また、組織独自のワークフローやブランディングを反映させるには、プラットフォームレベルのカスタマイズが必要

## いつ使うのか

外部SaaSを利用できない規制業界（金融、医療等）で自動化プラットフォームを構築したい時、組織独自のワークフローをノーコードで展開したい時、開発者とビジネスユーザーが協働する基盤が必要な時

## やり方

1. Activepiecesをオンプレミスまたはプライベートクラウドにデプロイ
2. ネットワークギャップ環境でも動作するよう構成（外部通信不要）
3. 開発者が独自pieceを開発し、社内ユーザーはノーコードビルダーで利用
4. ブランディング、アクセス制御、監査ログ等のエンタープライズ機能を有効化（Commercial License）
5. 組織全体で統一された自動化基盤として展開

### 入力

- オンプレミス/プライベートクラウド環境
- Commercial Licenseキー（エンタープライズ機能用）
- 組織独自のpiece開発

### 出力

- セルフホストActivepieces環境
- 組織専用の自動化ワークフロー
- ブランディング・アクセス制御されたプラットフォーム

## 使うツール・ライブラリ

- Activepieces（セルフホスト版）
- Docker/Kubernetes（デプロイ用）
- Commercial License

## 前提知識

- TypeScriptの基礎知識（型システム、非同期処理）
- npmパッケージ管理の理解
- REST API/Webhook等の統合パターンの知識
- Model Context Protocol (MCP)の基本概念（LLMがツールを呼び出す仕組み）
- ワークフロー自動化ツール（Zapier等）の利用経験があると理解が早い
