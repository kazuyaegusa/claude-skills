# 統合監視UIでエージェント状態を一元管理

> Electron製デスクトップアプリ（Superset）で複数worktree/エージェントのステータス、変更内容、通知を一箇所に集約し、ユーザーが必要なタイミングでのみ介入できるようにする

- 出典: https://github.com/superset-sh/superset
- 投稿者: superset-sh
- カテゴリ: agent-orchestration

## なぜ使うのか

複数のターミナルやエディタを切り替えると認知負荷が高く、エージェントの完了や質問を見逃しやすい。統合UIにより、エージェントが待機状態になったら通知を受け取り、diff確認・編集・承認を同じ画面で完結できる。

## いつ使うのか

3個以上のエージェントを並列実行する時、またはエージェント完了をリアルタイムで把握したい時

## やり方

1. Supersetアプリをインストール・起動
2. プロジェクトを開き、「New workspace」でworktreeベースのワークスペースを作成
3. 各ワークスペースで任意のCLIエージェントを起動（内蔵ターミナルから実行）
4. サイドバーに全ワークスペースがリスト表示され、エージェント状態（実行中/待機中/変更あり）がアイコンで可視化
5. エージェントが変更を完了すると通知が届く
6. 変更パネル（Changes panel）で差分を確認、必要に応じて内蔵エディタで編集
7. 承認後、`⌘O` でVSCode等の外部エディタに渡すか、そのままマージ操作

### 入力

- 複数のworktree
- 実行中のCLIエージェント群

### 出力

- 統合ダッシュボードでのステータス表示
- 変更通知とdiffビュー
- ワンクリックでのエディタ/ターミナル連携

## 使うツール・ライブラリ

- Superset（Electron, React, Tailwind）
- 内蔵diffビューア
- tRPC（フロント↔バックエンド通信）

## 前提知識

- git 2.20以上がインストールされていること（worktree機能）
- macOS環境（Windows/Linuxは未検証）
- Bun v1.0+ランタイム（ビルド・実行用）
- GitHub CLI（gh）とCaddy（開発サーバー用）のインストール
- CLIベースのAIエージェント（Claude Code, Codex, Gemini CLI等）が動作する環境
- git worktreeの基本概念（複数の作業ディレクトリを同一リポジトリで管理）の理解
