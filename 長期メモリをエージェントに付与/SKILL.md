# 長期メモリをエージェントに付与

> エージェントが過去のセッションや調査結果を永続ストレージに保存し、後続タスクで参照・利用できるようにする。

- 出典: https://x.com/dify_base/status/2031190239885353023
- 投稿者: AX Base| AX情報発信メディア
- カテゴリ: agent-orchestration

## なぜ使うのか

セッションをまたいでコンテキストを維持することで、同じ情報を再収集せず効率的に作業でき、複数ステップの自律タスクが実現できるため。

## いつ使うのか

複数日にまたがる調査タスクや、過去の判断結果を再利用したいワークフロー

### 具体的な適用場面

- コード生成・実行を含む複合タスクをAIに自律処理させたい場合
- 過去の調査結果や判断履歴をエージェントに持続的に参照させたいワークフロー構築時

## やり方

1. DeerFlowの `memory` コンポーネントを有効化（設定例: `memory.enabled: true`） 2. ベクトルDBまたはキーバリューストアを接続先として設定 3. エージェントが `remember(key, value)` / `recall(query)` ツールを自動的に呼び出してメモリを読み書きする 4. セッション間で `agent_id` を同一にすることで履歴が引き継がれる

### 入力

- agent_id（セッション識別子）
- 保存したいキー・バリューまたは検索クエリ

### 出力

- 過去セッションの保存済み情報・関連記憶の検索結果

## 使うツール・ライブラリ

- DeerFlow (bytedance/deer-flow)
- ベクトルDB（Chroma等）またはKVストア

## 前提知識

- Python 3.x および pip の基本操作
- Dockerの基本知識（サンドボックス実行を使う場合）
- LLMエージェント・ツール呼び出しの概念理解
- Claude Code CLI または Anthropic APIキー（Claude Code連携を使う場合）
