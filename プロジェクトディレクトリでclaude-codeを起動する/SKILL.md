# プロジェクトディレクトリでClaude Codeを起動する

> コードベースのルートディレクトリに移動してから `claude` コマンドを実行し、対話セッションを開始する

- 出典: https://github.com/anthropics/claude-code
- 投稿者: anthropics
- カテゴリ: claude-code-workflow

## なぜ使うのか

Claude Codeはカレントディレクトリをコンテキストとしてコードベース全体を理解するため、プロジェクトルートで起動することで適切なスコープでタスクを実行できる

## いつ使うのか

特定のプロジェクトに対してコード生成・リファクタリング・説明を依頼したい時

## やり方

1. `cd /path/to/your/project` でプロジェクトディレクトリに移動
2. `claude` コマンドを実行
3. 自然言語でタスクを指示（例: 'このコードベースのエントリーポイントを説明して'、'新しいAPIエンドポイントを追加して'）

### 入力

- プロジェクトディレクトリのパス
- 実行したいタスクの自然言語指示

### 出力

- コードの生成・編集・説明
- Git操作の実行結果
- ルーティンタスクの自動実行

## 使うツール・ライブラリ

- claude CLI

## コード例

```
cd ~/projects/my-app
claude
# セッション内で自然言語指示を入力
```

## 前提知識

- Node.js 18以上（実行環境として必要）
- ターミナル操作の基本知識
- Git の基本概念（Git操作機能を使う場合）
- 自然言語でタスクを指示する際の明確な表現力
