# ドメイン専用スキルで低品質コード生成を排除する

> ECC導入後はClaude Codeが依頼の内容からPyTorch・Next.js・フルスタック開発等のドメイン専用スキルを自動選択し、フレームワークのベストプラクティスに沿った高品質なコードを出力する

- 投稿者: 爱丽丝呀！
- カテゴリ: claude-code-workflow

## なぜ使うのか

汎用LLMは特定フレームワークの慣習・パターン・落とし穴を把握しきれず、低品質なコード（非推奨API使用・アンチパターン混入）を生成することがある。ドメイン特化知識を「スキル」として注入することでこの問題を根本から解消する

## いつ使うのか

特定フレームワーク（PyTorch・Next.js・React等）での実装時、AIが誤ったAPIや古いパターンを使うことが問題になっているとき

### 具体的な適用場面

- Claude Codeで複雑なプロジェクトを開発しており、コードレビューやセキュリティスキャンを手動でやっていて時間が取られているとき
- PyTorch・Next.js等の特定フレームワークでの実装中に、AIが一般的すぎる・品質の低いコードを出力してくることが問題になっているとき
- 長時間セッションでコンテキスト上限に達してAPI費用が嵩んでいるとき

## やり方

1. ECC導入後は追加操作不要。Claude Code上でコード生成を依頼するだけでスキルが自動適用される
2. 例：「PyTorchでResNetのファインチューニングコードを書いて」→ECC PyTorchスキルが自動起動し、DataLoader・Optimizer・学習ループをベストプラクティスで実装
3. 例：「Next.js App RouterでAPIルートを作って」→ECC Next.jsスキルがRSC・サーバーアクションの正しいパターンで生成
4. スキルの動作確認は `/list-skills` コマンドで利用可能スキル一覧を表示

### 入力

- 自然言語のコード生成依頼

### 出力

- ドメイン専用ベストプラクティスに沿ったコード
- フレームワーク固有のアンチパターンを含まない実装

## 使うツール・ライブラリ

- ECC Skills Library（PyTorch・Next.js・フルスタック等136スキル）

## 前提知識

- Claude Code または Cursor がインストールされていること
- Node.js（npx実行環境）が利用可能であること
- Claude Codeの基本操作（チャットでのコード依頼・スラッシュコマンド）を理解していること
