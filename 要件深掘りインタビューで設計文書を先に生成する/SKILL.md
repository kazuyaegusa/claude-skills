# 要件深掘りインタビューで設計文書を先に生成する

> AIエージェントがコードを書く前に質問フェーズを強制実行し、要件を完全に理解した上でユーザー承認済みの設計文書（SPEC）を生成する。

- 出典: https://x.com/jiroucaigou/status/2035964407201849564
- 投稿者: 努力赚钱的菜狗
- カテゴリ: agent-orchestration

## なぜ使うのか

曖昧な要件のままコードを書くとプロジェクト拡大時に全崩壊する。設計文書があれば後続のサブエージェントへの指示も明確になり、ズレが早期発見できる。

## いつ使うのか

新機能開発・新プロジェクト開始時、要件が頭の中にしかなくドキュメントが存在しない状態でAIに作業させるとき

### 具体的な適用場面

- AIが生成したコードは動くが、プロジェクト拡大後に保守不能になるパターンが繰り返されているとき
- 要件が曖昧なままAIに着手させて後から仕様ズレが発覚するケースを防ぎたいとき
- 複数の独立機能を並列開発してAIの作業スループットを上げたいとき
- AIエージェントに数時間の自律作業をさせたいが品質保証が不安なとき

## やり方

1. Superpowersをインストール後、コーディング依頼をするだけでエージェントが自動的に質問フェーズを開始する。2. エージェントは「何のために？」「誰が使う？」「成功状態は？」を聞いてくる。3. 回答後、読みやすいサイズに分割された設計文書をエージェントが提示する。4. ユーザーが承認してから実装フェーズへ移行。手動で同等のことをするには: `brainstorm` スキルを呼び出してから `write-plan` スキルで実装計画を生成する。

### 入力

- ユーザーからの自然言語での要望・アイデア

### 出力

- ユーザー承認済みの設計文書（SPEC）
- 実装計画（junior engineerでも追える粒度のタスクリスト）

## 使うツール・ライブラリ

- Superpowers
- Claude Code
- superpowers:brainstorm スキル
- superpowers:write-plan スキル

## 前提知識

- Claude Code CLI または Cursor がインストール済みであること
- Gitの基本操作（branch, merge, worktree）を理解していること
- TDD（テスト駆動開発）の赤→緑サイクルの概念を知っていること

## 根拠

> 「1.先疯狂提问，把你的需求完全理解，你同意后并生成标准设计文档」—— 要件深掘りと設計文書生成の強制

> 「2.开新分支+隔离工作区，确保不会搞乱主代码」—— ブランチ+隔離ワークスペースの使用

> 「3.把任务拆分成最小子任务」—— 最小サブタスク分割

> 「4.交给子Agent执行并强制测试，最小代码也要通过测试」—— サブエージェントへの委任とテスト強制

> 「5.对完成的任务进行双重审查，不仅要符合设计规格，更要检查代码质量，有一项不过关就不放行」—— 仕様適合＋コード品質のダブルレビュー
