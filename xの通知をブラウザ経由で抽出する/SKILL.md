# Xの通知をブラウザ経由で抽出する

> ブラウザでXを開き、最近の通知データを自動抽出する

- 出典: https://x.com/ix00ai/status/2035292820609925393
- 投稿者: iX
- カテゴリ: claude-code-workflow

## なぜ使うのか

X APIには制限があるため、ブラウザ操作経由で直接データを取得する方が柔軟

## いつ使うのか

X APIの利用枠を超えている時、またはAPI非公開の情報（自分専用の通知等）を取得したい時

### 具体的な適用場面

- Claude Codeでウェブスクレイピングを自動化したい時
- SNSの通知やメッセージを定期的に抽出・分析したい時
- ブラウザ操作を含む複雑なタスクをAIエージェントに任せたい時
- Playwright等の既存ツールで精度やスピードに不満がある時

## やり方

1. Browser Use CLI 2.0でXのURLを開く 2. ログイン状態を維持したまま通知ページへ遷移 3. 通知要素を特定して抽出（具体的なセレクタや抽出ロジックは投稿に記載なし）

### 入力

- XのログインセッションまたはCookie
- 通知ページのURL

### 出力

- 最近の通知データ（JSON形式等）

## 使うツール・ライブラリ

- Browser Use CLI 2.0

## 前提知識

- Claude Codeの基本的な使い方
- ブラウザ自動化の概念理解（Playwright, Puppeteer等）
- Bash/CLIツールの基本操作

## 根拠

> 「Claude Codeでブラウザ操作するなら、Browser Use CLI 2.0一択かも」

> 「スピード速く、精度も高そう」

> 「動画でやってるのは、・ブラウザで自分のXを開く ・最近の通知を抽出する」
