# Claude Codeセッション間チャット

> Claude Codeの複数セッション間でコンテキストを共有しながらタスク実行できる新機能の登場

- 出典: https://x.com/claudecode_lab/status/2035461982573539584
- 投稿者: Claude Code研究所|スパルタClaude Code塾
- カテゴリ: claude-code-workflow

## なぜ使うのか

複数のAIエージェントを協調させる場合、タスクの分散だけでなく途中経過や状態の共有が重要であり、セッション間チャット機能はエージェント協調の実用性を大きく向上させる

## いつ使うのか

長時間実行タスクを複数セッションに分割し、進捗を相互確認しながら進める場合, 異なる専門性を持つエージェント（例: フロントエンド担当とバックエンド担当）が協調して開発する場合

### 具体的な適用場面

- 長時間実行タスクを複数セッションに分割し、進捗を相互確認しながら進める場合
- 異なる専門性を持つエージェント（例: フロントエンド担当とバックエンド担当）が協調して開発する場合
- あるセッションで得た知見を別のセッションに即座に反映したい場合
- マルチエージェントシステムで中央調整役と実行役を分離する場合

## やり方

従来は独立したエージェント同士が協調する際にコンテキスト共有の手段が限られていたが、セッション同士が直接チャットでやりとりできるようになり、リアルタイムな情報交換が可能になった

## 前提知識

- Claude Codeの基本的な使い方
- 複数セッションを起動・管理する方法
- エージェント協調の基本概念

## 根拠

> Claude Codeセッション同士でのチャットやりとりを可能にする機能が登場

> それぞれ独立したエージェント同士でタスクをさせるたけでなく、途中途中でコンテキストの共有も可能になります
