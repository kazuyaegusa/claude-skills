# サブエージェント自動振り分けを設定する

> 入力タスクの種別（リサーチ/コーディング/要約など）を判定し、専門サブエージェントへ自動ルーティングする仕組みを持つ

- 出典: https://x.com/dify_base/status/2031190239885353023
- 投稿者: AX Base| AX情報発信メディア
- カテゴリ: agent-orchestration

## なぜ使うのか

汎用エージェント1つより専門エージェントへの振り分けの方が精度と効率が高い。オーケストレーターが判断するためユーザー側でタスク分類を書かなくてよい

## いつ使うのか

一つのパイプラインで調査・コード生成・レポート作成など複数種類のタスクが発生し、手動でサブエージェントを選ぶのが煩雑なとき

### 具体的な適用場面

- リサーチ・コード生成・データ分析など異種タスクが混在するAIワークフローを構築するとき
- エージェントにコードを実行させる必要があるが、ホストマシンを汚したくないとき
- 複数セッションにまたがる文脈（ユーザー履歴・過去の調査結果）をエージェントに保持させたいとき
- Claude Codeを既存のマルチエージェントパイプラインに組み込みたいとき

## やり方

1. `git clone https://github.com/bytedance/deer-flow && cd deer-flow` でリポジトリ取得
2. `pip install -r requirements.txt` で依存インストール
3. `conf.yaml` にLLMプロバイダーキーを設定（DeerFlowはOpenAI/Anthropic等に対応）
4. `python main.py --task 'あなたのタスク文字列'` で実行すると、オーケストレーターがタスクを解析して適切なサブエージェント（researcher/coder/reporter等）に振り分ける
5. ルーティングロジックは `src/agents/` 以下の各エージェント定義で確認・カスタマイズ可能

### 入力

- タスク記述文字列
- LLMプロバイダーのAPIキー（conf.yaml）

### 出力

- タスク種別に応じたサブエージェントの実行結果
- 統合レポート

## 使うツール・ライブラリ

- DeerFlow (bytedance/deer-flow)
- Python 3.10+

## 前提知識

- Python 3.10+ の実行環境
- LLMプロバイダー（OpenAI / Anthropic等）のAPIキー
- Docker（サンドボックス実行を使う場合）
- Node.js（Claude Code連携を使う場合）
- マルチエージェントアーキテクチャの基本概念（オーケストレーター・サブエージェント・ツール呼び出し）
