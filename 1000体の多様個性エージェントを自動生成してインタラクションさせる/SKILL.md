# 1000体の多様個性エージェントを自動生成してインタラクションさせる

> seed投入後に、異なる個性・記憶・行動パターンを持つ1000体のAIエージェントを自動生成し、Twitter形式の短文議論でインタラクションさせることで社会反応を創発させる。

- 出典: https://x.com/SuguruKun_ai/status/2033506937434947943
- 投稿者: すぐる | ChatGPTガチ勢 𝕏
- カテゴリ: agent-orchestration

## なぜ使うのか

均一エージェントでは現実の社会多様性を再現できない。個性・記憶・行動パターンを多様化することで、意見衝突→洗練→収束という現実に近いダイナミクスが生まれる。

## いつ使うのか

特定のテーマに対する多様な社会集団の反応分布を把握したいとき。単一の「代表的ユーザー像」では捉えられないマイノリティ意見や予想外の反発を発見したいとき。

### 具体的な適用場面

- 新政策・規制の施行前に世論の分極化・反発強度を事前シミュレーションしたいとき
- 金融市場への社会的イベント（地政学リスク・政策発表）の影響をエージェントベースでバックテストしたいとき
- 製品ローンチや広告キャンペーンへの多様な消費者反応を事前検証したいとき

## やり方

1. seedデータ投入後、MiroFishが自動で1000体分のエージェントプロファイルを生成
2. 各エージェントは独立したペルソナ（個性・バックグラウンド）、メモリ（信念・過去会話）、行動パターン（攻撃的/協調的など）を持つ
3. エージェント間でTwitter的な短文投稿形式の議論が自動展開
4. 意見がぶつかり→洗練され→収束するプロセスが自律的に進行
（※エージェント数1000体が固定値かパラメータかは投稿から確認不可）

### 入力

- seedテキスト
- エージェント数設定（デフォルト1000体）

### 出力

- 1000体分の個性・メモリ定義データ
- エージェント間議論ログ
- 意見収束結果と分布

## 使うツール・ライブラリ

- MiroFish (OSS)

## 前提知識

- Docker / docker-composeの基本操作
- GitHubからOSSをclone・実行できる開発環境
- シミュレーション対象のニュース・政策テキストの準備
- エージェントベースモデリング（ABM）の基礎概念の理解
