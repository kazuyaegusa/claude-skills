# 要件深掘りファーストで設計文書を先行生成する

> コードを書く前にAIが質問を重ねて要件を明確化し、ユーザーが承認した後に標準設計文書（spec）を生成するフローを強制する

- 出典: https://x.com/jiroucaigou/status/2035964407201849564
- 投稿者: 努力赚钱的菜狗
- カテゴリ: agent-orchestration

## なぜ使うのか

曖昧な要件のままAIに実装させると、プロジェクトが大きくなったときに全体崩壊する。ユーザーが承認した設計文書があることで、後のサブエージェントへの指示も一貫性を保てる

## いつ使うのか

新機能追加・バグ修正・リファクタリングなど、コーディングタスクをAIに依頼するとき全般

### 具体的な適用場面

- Claude Code / Cursorで新機能を実装しようとするたびにAIが設計なしに書き始めて後悔する状況
- チームでAIコーディングを使っているが、各人がバラバラなプロンプト戦略を取っていて品質が安定しない状況
- AIが生成したコードが動くが後のリファクタ・バグ修正が困難になっているプロジェクト

## やり方

SuperPowersのbrainstormingスキルが自動起動し、1. 「何のために作るか」「誰が使うか」「成功状態は何か」を聞く 2. 回答を受けてspec（設計文書）のドラフトを短い塊に分けて提示 3. ユーザーがOKを出したら次のステップへ進む。ユーザー側は追加操作不要

### 入力

- ユーザーの自然言語での依頼（曖昧でも可）

### 出力

- 承認済みの標準設計文書（spec）
- 次工程の実装計画生成への入力

## 使うツール・ライブラリ

- obra/superpowers (brainstormingスキル)

## 前提知識

- Claude Code または Cursor が動作する環境
- Node.js / npx が使える環境（CLIインストールの場合）
- git が使える環境（worktree機能のため）
- AIコーディングエージェントの基本的な使い方を知っていること
