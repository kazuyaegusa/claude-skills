# コンテキスト分散戦略（Background Agents）

> メインエージェントのコンテキストを軽く保つため、重い探索・検索タスクをバックグラウンドの専門エージェントに並列委譲する

- 出典: https://github.com/code-yeongyu/oh-my-openagent
- 投稿者: code-yeongyu
- カテゴリ: agent-orchestration

## なぜ使うのか

LLMのコンテキストウィンドウは有限。メインエージェントが全てを抱えると肥大化し、精度が落ちる。バックグラウンドで並列実行することでコンテキストを分散し、速度と精度を維持

## いつ使うのか

大規模コードベース、複数の情報源を並列検索したい場合、メインエージェントのコンテキストを節約したい場合

## やり方

1. Sisyphusがタスクを分析
2. ファイル検索、ドキュメント検索、コードベース探索などをExplore/Librarian等に並列委譲
3. 各エージェントが結果を返す
4. Sisyphusが統合して実装

### 入力

- 並列実行可能な複数のサブタスク

### 出力

- コンテキスト効率化
- 並列実行による高速化

## 使うツール・ライブラリ

- oh-my-opencode Background Agents機能

## 前提知識

- OpenCodeの基本的な使い方
- LLM API（Claude, GPT, Gemini等）のアクセス権
- npm/Node.js環境
- マルチエージェントオーケストレーションの概念理解
- LSP/ASTの基本的な理解（リファクタリング機能を使う場合）
