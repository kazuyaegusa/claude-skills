# 専門サブエージェントでタスクを役割分担実行

> ECCが提供する28の専門サブエージェント（TDD・セキュリティ・コードレビュー・ブレインストーミング等）をスラッシュコマンドで順次呼び出し、複合タスクを役割分担して処理する

- 出典: https://x.com/l_go_mrk/status/2037489559186145787
- 投稿者: AI駆動塾
- カテゴリ: claude-code-workflow

## なぜ使うのか

単一のClaude Codeで「コード生成・テスト・セキュリティチェック」を同時に高品質に処理することは困難だが、専門化されたサブエージェントが役割分担することで各工程の品質を独立して担保できる

## いつ使うのか

機能追加・バグ修正・リファクタリングが同時に絡む複合タスクで、単一プロンプトでの指示が曖昧になりがちな時

### 具体的な適用場面

- 新規プロジェクト開始時にClaude Codeのデフォルト能力をゼロから拡張したい場合
- PRマージ前・本番リリース前に多言語コードベースを横断してセキュリティ脆弱性を自動検出したい場合
- コード生成・テスト・セキュリティレビューを同一セッション内で並列処理したい場合

## やり方

1. ECCインストール後、各専門エージェントがスラッシュコマンドとして利用可能になる
2. 機能追加タスクの標準フロー:
   - `/brainstorm` → 設計案の発散
   - `/plan` → 実装計画の策定
   - `/tdd` → テストファーストで実装
   - `/security-review` → セキュリティ観点のレビュー
3. 各エージェントは独立したコンテキストで動作し、前段の出力を引き継いで処理する

### 入力

- ECCインストール済みのClaude Code環境
- タスク記述（自然言語で可）

### 出力

- ブレインストーミング結果・実装計画・テスト付きコード・セキュリティレポートの各工程成果物

## 使うツール・ライブラリ

- ecc-universal
- Claude Code Agent機能

## コード例

```
# 機能追加の標準フロー例
/brainstorm ユーザー認証APIの設計
/plan
/tdd
/security-review
```

## 前提知識

- Claude Code CLIがインストールされていること（`claude --version` で確認）
- Node.js / npxが使用可能な環境（ecc-universal/ecc-agentshieldのインストールに必要）
- GitHubアカウント（GitHub App経由のインストールを使う場合）

## 根拠

> 28の専門サブエージェントが連携して動く

> 125以上のSkillsをプリビルドで搭載

> 60以上のスラッシュコマンド

> セキュリティスキャン内蔵（102ルール、1,282テスト）

> 7言語対応
