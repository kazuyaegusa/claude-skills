# Cloudflare Scraping APIで全サイト自動爬取

> 単一URLを指定するだけで、サイト全体を自動巡回し、HTML/Markdown/JSONで取得

- 出典: https://x.com/xiaohu/status/2031921663751946646
- 投稿者: 小互
- カテゴリ: data-processing

## なぜ使うのか

従来の爬虫は各ページURLを手動収集・個別リクエストが必要。このAPIは自動でリンクを辿り、サイト構造全体を取得するため実装コストとメンテナンスコストを大幅削減

## いつ使うのか

複数ページにわたるサイト全体のデータ収集が必要で、爬虫インフラを自前構築したくない時

### 具体的な適用場面

- 競合サイトの価格・在庫情報を定期的に収集したい時
- ニュースサイト・ブログ全体をアーカイブしたい時
- React/Vue等のSPAサイトから構造化データを抽出したい時
- 変更検知機能を実装したいが、毎回全ページ爬取するとコストが高い時
- HTML解析を避け、直接JSON形式でデータ取得したい時

## やり方

1. Cloudflare APIキーを取得
2. APIエンドポイントに対象URL、出力形式（HTML/Markdown/JSON）、スキーマ定義を POST
3. レスポンスで構造化データまたは生HTMLを取得
4. 必要に応じてスキーマを定義し、JSON出力で特定フィールドのみ抽出

### 入力

- 対象サイトのURL（起点ページ）
- 出力形式指定（HTML/Markdown/JSON）
- （JSON出力時）取得したいデータのスキーマ定義

### 出力

- サイト全体のHTML/Markdown/JSON
- 構造化されたデータセット

## 使うツール・ライブラリ

- Cloudflare Scraping API

## 前提知識

- Cloudflareアカウントとアクティブなサブスクリプション（Scraping API利用可能プラン）
- HTTP APIの基本的な使い方（POST/GET、認証ヘッダー設定）
- JSONスキーマの基本理解（構造化データ抽出時）
- 対象サイトの利用規約確認（爬取が許可されているか）

## 根拠

> 给一个网址，自动爬完整站，返回HTML/Markdown/JSON

> 支持 JS 渲染（React/Vue 动态站也能爬）

> 增量爬取：只爬更新过的页面，省时省钱

> 要什么数据，直接返回结构化 JSON
