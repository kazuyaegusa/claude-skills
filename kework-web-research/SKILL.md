# KEWORK/AIXEED Webリサーチ・情報収集スキル

## 概要

KEWORKの業務判断に必要な情報をWebから収集し、構造化されたレポートとして出力する。
オフ会企画のコスト調査、取引先の会社情報確認、共有された動画やURLの要約など、実業務で発生するリサーチタスクに対応する。

## 実行手順

### 1. タスク分類

リサーチ依頼を受けたら、まず以下のカテゴリに分類する:

| カテゴリ | 判定基準 | テンプレート |
|---------|---------|------------|
| **会社・組織調査** | 会社名、法人名、組織名が含まれる | A. 会社リサーチ |
| **コスト・実現可能性調査** | 費用、コスト、実現可能性、日程が含まれる | B. コスト・フィージビリティ |
| **URL・コンテンツ確認** | URL、動画、Webサイト、ページが含まれる | C. コンテンツ確認 |
| **一般調査** | 上記に該当しない調査依頼 | D. 一般リサーチ |

### 2. 情報収集方法

#### Google AI Mode（推奨・第一選択）

google-ai-mode-skill がインストールされている場合は優先的に使用する。
ソース付きの包括的な調査結果が得られる。

```bash
# google-ai-mode-skill 経由で調査
python ~/.claude/skills/pleaseprompto--google-ai-mode-skill/scripts/run.py search.py \
  --query "{最適化したクエリ}" --save --debug
```

クエリ最適化のポイント:
- 日本語のタスクでも、英語圏の情報が必要なら英語クエリにする
- 年号（2026）を含めて最新情報を取得する
- 具体的な観点を括弧で列挙する

#### WebFetch（URL直接取得）

特定のURLが指定されている場合に使用する。

```
WebFetch ツールを使ってURLのコンテンツを取得する
```

#### WebSearch（キーワード検索）

google-ai-mode-skill が利用できない場合のフォールバック。

```
WebSearch ツールを使ってキーワード検索を行う
```

#### curl（API・構造化データ取得）

公開APIやJSON/XMLデータが必要な場合に使用する。

```bash
# Webページ取得（User-Agent指定）
curl -sL -A "Mozilla/5.0" "https://example.com" | head -200

# APIエンドポイント
curl -s "https://api.example.com/data" | python3 -m json.tool

# ヘッダ確認（リダイレクト先・ステータスコード）
curl -sI -L "https://example.com"
```

**調査項目**:

| 項目 | 調査方法 |

| 代表者 | 会社概要ページ、法人番号公表サイト |
| 所在地 | 会社概要ページ |
| 設立年 | 会社概要ページ |

#### C. コンテンツ確認テンプレート

**入力**: URL、動画リンク、Webサイト名

**調査項目**:
- ページ内容の要約
- 主要なメッセージ・主張
- KEWORKに関連する情報の抽出
- 動画の場合: タイトル、尺、内容の概要

```bash
# Webページのコンテンツ取得
# WebFetch ツールでURLを取得する

# YouTube動画の場合: メタ情報取得
curl -sL "https://www.youtube.com/oembed?url={動画URL}&format=json" | python3 -m json.tool

# OGP情報（タイトル・説明・サムネイル）取得
curl -sL -A "Mozilla/5.0" "{URL}" 2>/dev/null | python3 -c "
import sys, re
html = sys.stdin.read()
for tag in ['og:title', 'og:description', 'og:image', 'og:url']:
    m = re.search(f'<meta[^>]*property=\"{tag}\"[^>]*content=\"([^\"]+)\"', html)
    if m: print(f'{tag}: {m.group(1)}')
"
```

**出力フォーマット**:

```markdown
# {ページタイトル / 動画タイトル} 確認レポート

## エグゼクティブサマリー
- {内容の一言要約}
- {KEWORKとの関連性}
- {アクションが必要な場合はその内容}

## 確認元
- URL: {URL}
- 確認日: {日付}
- 種別: {Webページ / 動画 / PDF / SNS投稿}

## 内容詳細
{ページ・動画の内容を構造的にまとめる}

## KEWORKへの示唆
- {谷内への報告ポイント}
- {次のアクション提案}

## 情報ソース
- {参照したURL一覧}
```

### 4. 出力の共通ルール

すべてのレポートに共通するルール:

- **エグゼクティブサマリーは必ず3点以内**: 忙しい谷内が最初の3行で判断できるようにする
- **金額は日本円で統一**: 外貨の場合は `$1,000（約¥150,000 / 1USD=150JPY換算）` のように併記
- **情報ソースは必ず明記**: URLと確認日を記載。ソースなしの情報は `{要確認}` と明示する
- **推測と事実を明確に分ける**: 確認できなかった情報は `{未確認: 〜}` と明示する
- **古い情報に注意**: 価格・制度などは変動するため、情報の鮮度を注記する

### 6. 実タスク例と対応

| タスク | カテゴリ | テンプレート | 調査方法 |
|--------|---------|------------|---------|
| マレーシアオフ会の実現可能性・コスト・日程を調査する | コスト・フィージビリティ | B | Google AI Mode + WebSearch |
| 本田英樹氏が共有したプロモーション動画とアルファテクノロジー社のWebサイトを確認し、概要をまとめる | コンテンツ確認 + 会社調査 | C + A | WebFetch + Google AI Mode |
| クレジットカード会社に決済が通らない原因を問い合わせる | 一般調査 | D | WebSearch（FAQ確認）|
| FUSION AI社のサービス内容と料金体系を確認する | 会社調査 | A | WebFetch + Google AI Mode |
| 九州オフ会の会場候補を3つ比較する | コスト・フィージビリティ | B | Google AI Mode + WebSearch |

### 7. 複合タスクの処理

1つのタスクに複数カテゴリが含まれる場合（例: 「アルファテクノロジー社のWebサイトを確認し、概要をまとめる」→ コンテンツ確認 + 会社調査）:

1. 各カテゴリの調査を個別に実行する
2. レポートは1つにまとめ、セクションで区切る
3. エグゼクティブサマリーは全体を通した3点にまとめる

### 8. 注意事項

- **個人情報の取り扱い**: 調査対象の個人情報（住所、電話番号など）は、公開情報のみ記載する
- **著作権**: Webページの内容をそのまま転載しない。要約・引用にとどめる
- **調査の限界を明示する**: Webで得られる情報には限界がある。「この点はWeb調査では確認できませんでした。直接問い合わせが必要です」と明記する
- **アクセスできないコンテンツ**: ログイン必須のページ、有料コンテンツなどはその旨を報告する
- **問い合わせタスクの場合**: 問い合わせ先の電話番号・メールアドレス・営業時間を調べ、問い合わせ時のポイントを整理するところまでが本スキルの範囲。実際の電話・メール送信は人間が行う
