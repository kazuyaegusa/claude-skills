# サブエージェント自動振り分け

> オーケストレーターエージェントがタスクを解析し、検索・コード実行・分析等の専門サブエージェントへ自動でルーティングする。

- 出典: https://x.com/dify_base/status/2031190239885353023
- 投稿者: AX Base| AX情報発信メディア
- カテゴリ: agent-orchestration

## なぜ使うのか

タスク種別ごとに最適なエージェントを使い分けることで、汎用プロンプト1つより精度・効率が上がるため。

## いつ使うのか

検索・コード実行・ドキュメント要約など複数の異なる能力が必要なタスクを一度に処理したいとき

### 具体的な適用場面

- コード生成・実行を含む複合タスクをAIに自律処理させたい場合
- 過去の調査結果や判断履歴をエージェントに持続的に参照させたいワークフロー構築時

## やり方

1. DeerFlowリポジトリをクローン: `git clone https://github.com/bytedance/deer-flow` 2. 依存をインストール: `pip install -r requirements.txt` 3. オーケストレーター設定ファイルでサブエージェントの種別と担当タスクを定義 4. `python main.py --task "調査タスクのテキスト"` で起動するとオーケストレーターがサブエージェントを選択して実行する

### 入力

- タスク指示テキスト
- サブエージェント定義設定ファイル

### 出力

- 各サブエージェントの実行結果を統合した最終回答

## 使うツール・ライブラリ

- DeerFlow (bytedance/deer-flow)
- Python 3.x

## 前提知識

- Python 3.x および pip の基本操作
- Dockerの基本知識（サンドボックス実行を使う場合）
- LLMエージェント・ツール呼び出しの概念理解
- Claude Code CLI または Anthropic APIキー（Claude Code連携を使う場合）
