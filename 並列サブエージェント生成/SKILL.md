# 並列サブエージェント生成

> 独立したサブエージェントを並列起動し、マルチストリーム処理を実現。PythonスクリプトからRPC経由でツール呼び出し可能。

- 出典: https://github.com/NousResearch/hermes-agent
- 投稿者: NousResearch
- カテゴリ: agent-orchestration

## なぜ使うのか

複雑なタスクを直列処理すると遅延が大きい。並列化することで、調査・実装・テストを同時進行し、全体時間を短縮できる。

## いつ使うのか

調査・実装・テストを同時並行したい場合、マルチステップパイプラインを効率化したい場合

## やり方

1. エージェントが並列可能なサブタスクを検出 2. 独立したサブエージェントを起動 3. 各サブエージェントが並列実行 4. PythonスクリプトからRPC経由でツール呼び出しして、マルチステップパイプラインをコンテキストコストゼロで実行

### 入力

- 並列可能なサブタスク

### 出力

- 並列実行結果
- 短縮された実行時間

## 使うツール・ライブラリ

- hermes subagent
- Python RPC

## 前提知識

- 基本的なコマンドライン操作
- LLM APIキー（OpenRouter/OpenAI/Anthropic等のいずれか）
- gitがインストールされた環境
- Linux/macOS/WSL2のいずれか
