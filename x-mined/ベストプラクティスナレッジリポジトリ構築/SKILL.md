# ベストプラクティスナレッジリポジトリ構築

> プログラムコードではなく、ツール活用ナレッジ・ベストプラクティス・Tips・ワークフローをマークダウンで構造化・集約したリポジトリを作成・維持する。

- 出典: https://x.com/labelmake/status/2032652943250338098
- 投稿者: Kyohei - OSS, 外資IT
- カテゴリ: claude-code-workflow

## なぜ使うのか

AIエージェント時代において最も高レバレッジなのはコード実装ではなくナレッジの構造化。このリポジトリ自体がGitHubトレンド入りしたことがその証拠であり、「マークダウンがプログラムよりレバレッジが高い」という時代の転換を体現している。

## いつ使うのか

特定ツール・フレームワークを組織・チームで本格活用し始める時、コミュニティへの知識貢献を考えている時

### 具体的な適用場面

- Claude Codeを業務・開発に本格導入する際に、Commands/Agents/Skills/Hooksの使い分けを体系的に学びたい場面
- チームでClaude Code運用ルールを標準化し、CLAUDE.mdやサブエージェント設計のベースラインを作りたい場面
- 自分のワークフローを見直してコンテキスト管理・並列開発・プランモードを効果的に活用したい場面

## やり方

1. ツール・手法のCONCEPTSセクションを表形式で整理（機能名/場所/説明）
2. 公式ドキュメントと実践的なコミュニティTipsを分けて収録
3. 各テクニックに「best-practice」「implemented」などのバッジで状態を明示
4. Hot/新着セクションを設けて最新情報を優先表示
5. 実装例・デモ・GIFを同梱して理解を加速
6. 「Billion-Dollar Questions」として未解決の重要問いを記録
7. コミュニティからのフィードバック・ワークフローで自動更新

### 入力

- 公式ドキュメント
- コミュニティTips・X投稿
- 自身の実践経験

### 出力

- 構造化されたベストプラクティス集
- GitHubリポジトリ（スター・フォーク・トレンド入り可能）
- チームのオンボーディング資料

## 使うツール・ライブラリ

- GitHub
- Markdown
- シールドバッジ（shields.io）
- 自動更新ワークフロー

## 根拠

> 「公式・コミュニティのClaude Codeベストプラクティスを1箇所にまとめたリポジトリがGitHub月間トレンド入り」

> 「プログラムではなく、情報をまとめたマークダウンがトレンド入りするのがまさに今っぽい」

> 「最もレバレッジが効く場所がプログラムではなく、ナレッジになった感」

> 「avoid agent dumb zone, do manual /compact at max 50%. Use /clear to reset context mid-session if switching to a new task」

> 「always start with plan mode (Boris)」
