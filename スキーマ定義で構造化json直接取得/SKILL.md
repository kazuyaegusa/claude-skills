# スキーマ定義で構造化JSON直接取得

> HTMLパース不要。APIリクエスト時にスキーマを渡し、指定フィールドのみJSON形式で取得

- 出典: https://x.com/xiaohu/status/2031921663751946646
- 投稿者: 小互
- カテゴリ: data-processing

## なぜ使うのか

従来はHTML取得後、BeautifulSoup等でパース・抽出が必要。スキーマ定義により、API側で抽出・構造化を完結し、クライアント側の実装を不要に

## いつ使うのか

HTMLパース実装を書きたくない時、または抽出ロジックをAPIに委譲してメンテナンスコストを下げたい時

### 具体的な適用場面

- 競合サイトの価格・在庫情報を定期的に収集したい時
- ニュースサイト・ブログ全体をアーカイブしたい時
- React/Vue等のSPAサイトから構造化データを抽出したい時
- 変更検知機能を実装したいが、毎回全ページ爬取するとコストが高い時
- HTML解析を避け、直接JSON形式でデータ取得したい時

## やり方

1. 取得したいデータ構造をJSONスキーマで定義（例: {"title": "string", "price": "number", "image": "url"}）
2. API呼び出し時、出力形式JSONとスキーマを指定
3. APIは各ページから該当フィールドを抽出し、配列形式のJSONで返す
4. クライアントは返されたJSONをそのまま利用

### 入力

- 取得したいデータ構造のJSONスキーマ定義

### 出力

- スキーマに従った構造化JSON（配列形式）

## 使うツール・ライブラリ

- Cloudflare Scraping API（JSON出力+スキーマ指定）

## 前提知識

- Cloudflareアカウントとアクティブなサブスクリプション（Scraping API利用可能プラン）
- HTTP APIの基本的な使い方（POST/GET、認証ヘッダー設定）
- JSONスキーマの基本理解（構造化データ抽出時）
- 対象サイトの利用規約確認（爬取が許可されているか）

## 根拠

> 给一个网址，自动爬完整站，返回HTML/Markdown/JSON

> 支持 JS 渲染（React/Vue 动态站也能爬）

> 增量爬取：只爬更新过的页面，省时省钱

> 要什么数据，直接返回结构化 JSON
